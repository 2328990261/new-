# 教师注册失败修复指南

## 问题描述

提交教师注册时失败，数据包含新字段但数据库表中没有对应的列。

## 原因

数据库 `teacher` 表缺少新增的字段：
- `hometown` - 籍贯
- `teaching_years` - 教龄
- `birth_year` - 出生年份
- `location_province` - 所在省份
- `location_city` - 所在城市
- `location_district` - 所在区县
- `location_address` - 详细地址
- `self_intro` - 自我介绍

## 解决方案

### 方法一：自动修复（推荐）

在后端目录执行：

```bash
cd tjiajiao91/backend
php fix_teacher_fields.php
```

这个脚本会：
1. 检查哪些字段缺失
2. 自动添加缺失的字段
3. 添加索引
4. 显示执行结果

### 方法二：手动执行SQL

在数据库中执行SQL文件：

```bash
mysql -u用户名 -p 数据库名 < tjiajiao91/database/add_teacher_location_fields.sql
```

或者在phpMyAdmin/Navicat等工具中直接执行SQL：

```sql
-- 添加字段
ALTER TABLE `teacher` 
ADD COLUMN `hometown` VARCHAR(100) NULL COMMENT '籍贯' AFTER `email`,
ADD COLUMN `teaching_years` INT(11) NULL DEFAULT 0 COMMENT '教龄（年）' AFTER `hometown`,
ADD COLUMN `birth_year` INT(11) NULL COMMENT '出生年份' AFTER `teaching_years`,
ADD COLUMN `location_province` VARCHAR(50) NULL COMMENT '所在省份' AFTER `birth_year`,
ADD COLUMN `location_city` VARCHAR(50) NULL COMMENT '所在城市' AFTER `location_province`,
ADD COLUMN `location_district` VARCHAR(50) NULL COMMENT '所在区县' AFTER `location_city`,
ADD COLUMN `location_address` VARCHAR(200) NULL COMMENT '详细地址' AFTER `location_district`;

-- 添加自我介绍字段
ALTER TABLE `teacher`
ADD COLUMN `self_intro` TEXT NULL COMMENT '自我介绍' AFTER `experience`;

-- 添加索引
ALTER TABLE `teacher` 
ADD INDEX `idx_hometown` (`hometown`),
ADD INDEX `idx_location_city` (`location_city`),
ADD INDEX `idx_teaching_years` (`teaching_years`),
ADD INDEX `idx_birth_year` (`birth_year`);
```

### 方法三：检查字段

如果不确定是否需要修复，先检查：

```bash
cd tjiajiao91/backend
php check_teacher_fields.php
```

这会显示：
- 当前所有字段
- 哪些新字段已存在
- 哪些新字段缺失

## 验证修复

修复后，再次尝试注册教师，应该可以成功。

## 注意事项

1. **备份数据库** - 执行SQL前建议备份数据库
2. **权限问题** - 确保数据库用户有ALTER TABLE权限
3. **字段顺序** - SQL中指定了字段的插入位置（AFTER）
4. **索引** - 添加了索引以提高查询性能
5. **NULL值** - 所有新字段都允许NULL，不影响现有数据

## 常见错误

### 错误1：字段已存在
```
Duplicate column name 'hometown'
```
**解决**：字段已经存在，无需重复添加

### 错误2：权限不足
```
Access denied for user
```
**解决**：使用有ALTER权限的数据库用户

### 错误3：表不存在
```
Table 'teacher' doesn't exist
```
**解决**：检查数据库名称和表名是否正确
