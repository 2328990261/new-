# 教师字段显示问题修复说明

## 问题描述
管理后台教师详情页面中，以下字段没有显示：
- 出生年月
- 邮箱
- OpenID
- 微信昵称

## 问题原因
Teacher 模型中的字段定义与数据库实际字段不匹配：
- 模型中定义为 `birth_date`，但数据库字段是 `birth_year`

## 修复内容

### 1. 修改 Teacher 模型
文件：`new_system/backend/app/model/Teacher.php`

将字段定义从 `birth_date` 改为 `birth_year`：
```php
protected $schema = [
    // ...
    'birth_year'            => 'int',  // 原来是 'birth_date' => 'string'
    // ...
];
```

### 2. 修改 Teacher 控制器
文件：`new_system/backend/app/controller/admin/Teacher.php`

更新允许更新的字段列表：
```php
$allowedFields = [
    'name', 'gender', 'phone', 'wechat_id', 'wechat_nickname', 'openid', 'email',
    'hometown', 'teaching_years', 'birth_year',  // 原来是 birth_date
    // ...
];
```

### 3. 数据库字段检查
创建了 SQL 脚本 `check_and_add_teacher_fields.sql` 来检查并添加缺失的字段。

## 注意事项

1. **字段类型**：
   - `birth_year` 是整数类型（INT），存储年份如 1990
   - 如果需要存储完整日期，应该使用 DATE 类型

2. **数据迁移**：
   - 如果之前使用了 `birth_date` 字段，需要迁移数据到 `birth_year`
   - 可以使用 SQL 提取年份：`UPDATE fa_teachers SET birth_year = YEAR(birth_date)`

3. **前端显示**：
   - 邮箱字段受"显示联系方式"按钮控制
   - OpenID 和微信昵称直接显示

4. **数据来源**：
   - 这些字段主要来自微信小程序登录和教师注册流程
   - 如果教师没有通过小程序注册，这些字段可能为空
