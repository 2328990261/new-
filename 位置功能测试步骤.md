# 位置信息功能测试步骤

## 第一步：执行数据库迁移

### 方法一：使用迁移脚本（推荐）

1. 在浏览器中访问：
   ```
   https://t.jiajiao91.com/api/migrate_user_location.php
   ```

2. 页面会显示迁移进度和结果
   - 如果显示"✓ 迁移完成！"，说明迁移成功
   - 如果显示"⚠ 字段已存在"，说明已经迁移过了
   - 如果显示错误信息，请检查数据库配置

3. 迁移成功后，页面会显示新添加的字段和索引

### 方法二：在phpMyAdmin中手动执行

1. 登录phpMyAdmin
2. 选择数据库 `myjiajiao`
3. 点击"SQL"标签
4. 复制并执行以下SQL：

```sql
-- 为微信用户表添加位置信息字段
ALTER TABLE `fa_wechat_users`
ADD COLUMN `latitude` decimal(10,7) DEFAULT NULL COMMENT '纬度' AFTER `user_id`,
ADD COLUMN `longitude` decimal(10,7) DEFAULT NULL COMMENT '经度' AFTER `latitude`,
ADD COLUMN `address` varchar(500) DEFAULT NULL COMMENT '详细地址' AFTER `longitude`,
ADD COLUMN `province` varchar(50) DEFAULT NULL COMMENT '省份' AFTER `address`,
ADD COLUMN `city` varchar(50) DEFAULT NULL COMMENT '城市' AFTER `province`,
ADD COLUMN `district` varchar(50) DEFAULT NULL COMMENT '区县' AFTER `city`;

-- 添加位置索引以提高查询性能
ALTER TABLE `fa_wechat_users`
ADD INDEX `idx_location` (`latitude`, `longitude`),
ADD INDEX `idx_city` (`city`, `district`);
```

## 第三步：在小程序中测试完整流程

### 1. 准备测试环境
- 确保小程序已编译并上传到微信开发者工具
- 确保数据库迁移已完成
- 确保API文件已上传到服务器

### 2. 测试登录流程
1. 在小程序中点击"退出登录"（如果已登录）
2. 重新进入登录页面
3. 设置头像和昵称
4. 勾选用户协议
5. 点击"手机号快捷登录"
6. 授权手机号

### 3. 测试位置授权
登录成功后应该看到：
1. 弹出位置授权说明弹窗
   - 标题："获取你的位置信息"
   - 内容："你的位置信息将用于为你推荐附近的优质家教老师"
   - 按钮："允许" 和 "拒绝"

2. 点击"允许"后：
   - 打开微信的位置选择页面
   - 选择一个位置
   - 自动保存位置信息

3. 点击"拒绝"：
   - 不影响登录流程
   - 正常跳转到下一页

### 4. 验证位置信息已保存

在数据库中查询：
```sql
SELECT 
  openid, 
  nickname, 
  phone,
  latitude, 
  longitude, 
  address, 
  province, 
  city, 
  district,
  update_time
FROM fa_wechat_users 
WHERE phone = '你的手机号'
ORDER BY update_time DESC
LIMIT 1;
```

应该能看到：
- latitude 和 longitude 有值
- address、province、city、district 有值
- update_time 是最新的时间

## 第四步：测试边界情况

### 1. 用户拒绝位置授权
- 登录成功
- 点击"拒绝"
- 应该正常跳转，不影响使用

### 2. 位置保存失败
- 检查控制台是否有错误日志
- 确认不影响登录流程

### 3. 重复登录
- 第二次登录时不应该再次请求位置授权
- 除非用户主动更新位置

## 常见问题排查

### 1. 迁移脚本访问404
- 检查文件是否上传到服务器
- 检查文件路径是否正确：`api/migrate_user_location.php`

### 2. 位置保存失败
- 检查数据库连接配置
- 检查用户是否存在（openid是否正确）
- 检查字段是否已添加

### 3. 小程序不显示位置授权弹窗
- 检查登录流程是否成功
- 检查 `location-auth.js` 文件是否正确导入
- 检查控制台是否有错误

### 4. 位置选择后没有保存
- 检查API地址是否正确
- 检查网络请求是否成功
- 检查控制台错误日志
