# 卡包功能说明

## 功能概述

卡包页面用于展示和管理用户通过邀请好友获得的优惠券。用户可以在这里查看、领取和使用优惠券。

## 页面结构

### 1. 统计卡片
显示用户优惠券的整体情况：
- 优惠券总数
- 可用优惠券（待领取+已领取）
- 已使用优惠券

### 2. 标签页
四个标签页分别显示不同状态的优惠券：
- **待领取**：邀请好友认证成功后，系统发放的优惠券，需要用户手动领取
- **已领取**：用户已领取到卡包，可以使用的优惠券
- **已使用**：管理员已兑换的优惠券
- **已过期**：超过有效期的优惠券

### 3. 优惠券卡片
每张优惠券显示以下信息：
- 优惠券金额（￥20）
- 优惠券类型（邀请奖励券）
- 优惠券标题（邀请好友奖励/新用户注册奖励）
- 来源（邀请好友获得/被邀请注册获得）
- 时间信息（创建/领取/使用/过期时间）
- 有效期
- 操作按钮（立即领取/已领取/已使用/已过期）

## 优惠券状态流转

```
待领取(0) -> 已领取(1) -> 已使用(2)
                    |
                    v
                已过期(3)
```

### 状态说明

1. **待领取 (status=0)**
   - 邀请的好友认证成功后，系统自动发放
   - 显示"立即领取"按钮
   - 用户点击后变为"已领取"状态

2. **已领取 (status=1)**
   - 用户已领取到卡包
   - 显示"已领取"标签
   - 等待管理员兑换

3. **已使用 (status=2)**
   - 管理员在后台进行人工兑换
   - 显示"已使用"标签
   - 卡片半透明显示
   - 显示使用时间

4. **已过期 (status=3)**
   - 超过有效期（默认30天）未使用
   - 显示"已过期"标签
   - 卡片半透明显示
   - 显示过期时间

## 个人中心入口

在个人中心页面添加了"我的卡包"入口：
- 显示卡包图标（🎫）
- 显示可用优惠券数量徽章
- 点击进入卡包页面

## API接口

### 获取我的优惠券
```
GET /api/invitation/my-coupons
参数：
  - status: 可选，筛选状态（0/1/2/3）
返回：
  - 优惠券列表
```

### 领取优惠券
```
POST /api/invitation/receive-coupon
参数：
  - coupon_id: 优惠券ID
返回：
  - 领取结果
```

## 使用流程

### 用户端流程

1. **获得优惠券**
   - 用户A邀请用户B注册
   - 用户B完成认证
   - 系统自动为用户A和用户B各发放一张￥20优惠券（状态：待领取）

2. **领取优惠券**
   - 用户进入"我的卡包"
   - 在"待领取"标签页看到新优惠券
   - 点击"立即领取"按钮
   - 优惠券状态变为"已领取"

3. **使用优惠券**
   - 用户联系客服或管理员
   - 管理员在后台搜索用户
   - 管理员进行人工兑换
   - 优惠券状态变为"已使用"

### 管理端流程

1. **查看优惠券**
   - 进入"邀请管理"页面
   - 切换到"优惠券管理"标签
   - 查看所有用户的优惠券

2. **搜索用户优惠券**
   - 点击"搜索用户优惠券"按钮
   - 输入用户昵称或手机号
   - 查看该用户的所有优惠券

3. **兑换优惠券**
   - 找到需要兑换的优惠券（状态：已领取）
   - 点击"兑换"按钮
   - 输入兑换备注
   - 确认兑换
   - 优惠券状态变为"已使用"

4. **批量兑换**
   - 勾选多张优惠券
   - 点击"批量兑换"按钮
   - 确认批量兑换

## 注意事项

1. **优惠券有效期**
   - 默认30天有效期
   - 过期后自动变为"已过期"状态
   - 已过期的优惠券无法使用

2. **领取限制**
   - 每张优惠券只能领取一次
   - 已领取的优惠券不能重复领取

3. **兑换限制**
   - 只有"已领取"状态的优惠券可以兑换
   - 兑换后无法撤销

4. **显示优化**
   - 已使用和已过期的优惠券显示为半透明
   - 可用优惠券数量实时更新
   - 个人中心显示可用优惠券数量徽章

## 水印图片
