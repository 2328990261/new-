# 定位功能说明

## 功能概述
教师注册页面的"所在区域"功能使用微信的位置选择API (`uni.chooseLocation`)，让用户可以通过地图选择位置，然后自动解析为省市区。

## 实现流程

### 1. 用户点击"点击定位"按钮
```javascript
handleGetLocation()
```

### 2. 调用微信位置选择API
```javascript
uni.chooseLocation({
  success: (location) => {
    // location包含：
    // - name: 位置名称
    // - address: 详细地址
    // - latitude: 纬度
    // - longitude: 经度
  }
})
```

### 3. 将经纬度发送到后端API进行逆地理编码
```javascript
// 调用后端API
regionApi.reverseGeocode(latitude, longitude)

// 后端调用腾讯地图API
// GET /api/geocode/reverse?latitude=xxx&longitude=xxx
```

### 4. 后端返回标准化的省市区数据
```json
{
  "code": 200,
  "data": {
    "province": "广东省",
    "city": "广州市",
    "district": "越秀区",
    "formatted_address": "广东省广州市越秀区府前路"
  }
}
```

### 5. 前端匹配数据库中的省市区
```javascript
matchAndSetLocation(addressParts)
```
- 查找匹配的省份
- 加载该省份下的城市
- 查找匹配的城市
- 加载该城市下的区县
- 查找匹配的区县
- 设置picker的值

### 6. 用户手动输入详细地址
**注意：详细地址不会自动填充，用户必须手动输入**

## 关键代码位置

### 前端
- **文件**: `miniprogram/pages/teacher-register/index.vue`
- **方法**:
  - `handleGetLocation()` - 处理定位按钮点击
  - `parseAddressViaAPI()` - 调用后端API解析地址
  - `matchAndSetLocation()` - 匹配并设置位置

### 后端
- **文件**: `tjiajiao91/backend/app/controller/api/Geocode.php`
- **方法**: `reverse()` - 逆地理编码
- **路由**: `GET /api/geocode/reverse`

### API工具
- **文件**: `miniprogram/utils/api.js`
- **方法**: `regionApi.reverseGeocode()`

## 配置要求

### 后端环境变量
在 `.env` 文件中配置腾讯地图API密钥：
```
TENCENT_MAP_KEY=your_tencent_map_key_here
```

### 小程序权限配置
在 `manifest.json` 中添加位置权限：
```json
{
  "mp-weixin": {
    "permission": {
      "scope.userLocation": {
        "desc": "您的位置信息将用于自动填写所在区域"
      }
    },
    "requiredPrivateInfos": [
      "chooseLocation"
    ]
  }
}
```

## 用户体验流程

1. 用户点击"点击定位"按钮
2. 微信打开地图选择界面
3. 用户可以：
   - 搜索地址
   - 在地图上选择位置
   - 查看附近的POI（兴趣点）
4. 选择完成后，系统自动解析并填充省市区
5. 用户手动输入详细地址（街道、门牌号等）

## 优势

### 相比 `uni.getLocation`
- ✅ 用户可以主动选择位置，而不是被动获取当前位置
- ✅ 可以搜索任意地址
- ✅ 可以在地图上精确选择
- ✅ 返回的地址信息更完整（包含name和address）
- ✅ 用户体验更好

### 相比前端解析地址字符串
- ✅ 使用腾讯地图API，解析更准确
- ✅ 支持各种地址格式
- ✅ 返回标准化的省市区数据
- ✅ 避免前端正则匹配的不准确性

## 注意事项

1. **详细地址不自动填充**
   - 只自动填充省市区
   - 详细地址需要用户手动输入
   - 这样可以确保地址的准确性

2. **需要腾讯地图API密钥**
   - 在腾讯地图开放平台申请
   - 配置在后端 `.env` 文件中
   - 如果没有配置，会返回简单的经纬度信息

3. **用户可以取消选择**
   - 如果用户取消选择，不会显示错误提示
   - 用户可以手动使用picker选择省市区

4. **匹配可能失败**
   - 如果后端返回的省市区名称与数据库不匹配
   - 会提示用户手动选择
   - 这种情况很少见，因为使用的是标准地名

## 常见问题

### Q: 为什么不使用 `uni.getLocation`？
A: `uni.getLocation` 只能获取当前位置的经纬度，用户无法选择其他位置。而 `uni.chooseLocation` 允许用户在地图上搜索和选择任意位置，体验更好。

### Q: 为什么要调用后端API解析地址？
A: 腾讯地图API的逆地理编码功能更准确，可以返回标准化的省市区数据。前端直接解析地址字符串容易出错。

### Q: 详细地址为什么不自动填充？
A: 为了确保地址的准确性。自动填充的详细地址可能不够精确，让用户手动输入可以确保地址正确。

### Q: 如果没有配置腾讯地图API密钥会怎样？
A: 后端会返回简单的经纬度信息，前端会提示用户手动选择省市区。功能不会完全失效。
