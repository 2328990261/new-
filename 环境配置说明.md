# 环境配置说明

## 重要提示

本项目的配置文件（`.env` 和 `config/database.php`）因包含敏感信息，已在 `.gitignore` 中排除，不会提交到Git仓库。

## 首次部署配置步骤

### 1. 后端配置

#### 创建 .env 文件
```bash
cd backend
cp .env.example .env
```

然后编辑 `.env` 文件，确保以下配置正确：

```env
# 数据库配置
DB_HOST = 127.0.0.1
DB_PORT = 3306
DB_DATABASE = myjiajiao
DB_USERNAME = jE2se7DGe5HfE6zL
DB_PASSWORD = myjiajiao

# 重要：数据库表前缀必须设置
database.prefix = fa_
```

#### 创建 database.php 文件
```bash
cd backend/config
cp database.example.php database.php
```

确保 `database.php` 中的表前缀设置为 `fa_`：
```php
'prefix' => env('database.prefix', 'fa_'),
```

### 2. 安装依赖

#### 后端依赖
```bash
cd backend
composer install --no-dev
```

#### 前端依赖
```bash
# 管理端
cd frontend/admin
npm install

# 用户端
cd frontend/user
npm install
```

### 3. 数据库导入

确保已导入完整的数据库SQL文件，数据库中的表应该都带有 `fa_` 前缀，例如：
- `fa_admin`
- `fa_teacher`
- `fa_tutor_order`
- 等等

### 4. 启动服务

#### 后端
```bash
cd backend/public
php -S localhost:8080 router.php
```

#### 前端
```bash
# 管理端（终端1）
cd frontend/admin
npm run dev

# 用户端（终端2）
cd frontend/user
npm run dev
```

## 常见问题

### Q1: 登录时提示 "Table 'myjiajiao.admin' doesn't exist"

**原因**: 数据库表前缀配置错误

**解决方案**:
1. 检查 `.env` 文件中是否有 `database.prefix = fa_`
2. 检查 `config/database.php` 中的 prefix 配置
3. 重启后端服务

### Q2: 依赖安装失败

**后端依赖问题**:
```bash
composer clear-cache
composer install --no-dev
```

**前端依赖问题**:
```bash
rm -rf node_modules package-lock.json
npm install
```

### Q3: Git切换分支后配置丢失

这是正常现象，因为配置文件在 `.gitignore` 中。解决方案：

**方法1**: 切换分支前备份
```bash
cp backend/.env backend/.env.backup
cp backend/config/database.php backend/config/database.php.backup
```

**方法2**: 使用示例文件快速恢复
```bash
cp backend/.env.example backend/.env
cp backend/config/database.example.php backend/config/database.php
# 然后编辑配置文件
```

## 配置文件清单

### 必须手动创建的文件
- ✅ `backend/.env` - 环境配置（从 .env.example 复制）
- ✅ `backend/config/database.php` - 数据库配置（从 database.example.php 复制）

### 自动生成的目录
- `backend/vendor/` - Composer依赖
- `backend/runtime/` - 运行时缓存
- `frontend/admin/node_modules/` - 前端依赖
- `frontend/user/node_modules/` - 前端依赖

## 安全提示

⚠️ **重要**: 
1. 永远不要将 `.env` 和 `database.php` 提交到Git
2. 生产环境的密码要使用强密码
3. 定期备份数据库
4. 生产环境关闭调试模式（`APP_DEBUG = false`）
