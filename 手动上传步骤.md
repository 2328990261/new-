# 手动上传步骤（文件被占用时）

## 问题说明

当前有些文件被其他程序占用，导致 Git 无法添加文件。这通常是因为：
- 编辑器（VS Code/Kiro）正在打开这些文件
- PHP 进程正在运行
- 文件资源管理器正在访问这些目录

## 解决方案

### 方法 1: 关闭占用程序后重试（推荐）

1. **关闭所有可能占用文件的程序**：
   - 关闭 VS Code / Kiro IDE
   - 停止所有 PHP 进程（如果有运行 `启动服务.bat`）
   - 关闭文件资源管理器中的 new_system 目录

2. **重新打开命令行窗口**：
   ```bash
   cd e:\123.207.52.46_8088\new_system
   ```

3. **执行上传命令**：
   ```bash
   git add .
   git commit -m "feat: 91家教管理系统初始版本"
   git push -u origin main
   ```

### 方法 2: 使用批处理脚本

1. **关闭所有程序**（同上）

2. **双击运行**：
   ```
   上传到Gitee.bat
   ```

### 方法 3: 分步添加文件

如果仍然有问题，可以分步添加：

```bash
cd e:\123.207.52.46_8088\new_system

# 添加文档文件
git add *.md *.txt .gitignore

# 添加配置文件
git add *.conf *.bat

# 添加数据库文件
git add database/

# 添加后端代码（排除被占用的文件）
git add backend/app/
git add backend/config/
git add backend/route/
git add backend/public/
git add backend/composer.json

# 添加前端代码
git add frontend/

# 提交
git commit -m "feat: 91家教管理系统初始版本"

# 推送
git push -u origin main
```

### 方法 4: 使用 Git GUI 工具

1. 下载安装 **TortoiseGit** 或 **GitHub Desktop**

2. 打开 `e:\123.207.52.46_8088\new_system` 目录

3. 使用图形界面进行提交和推送

## 检查哪些文件被占用

运行以下命令查看：

```bash
# 在 PowerShell 中
Get-Process | Where-Object {$_.Path -like "*123.207.52.46_8088*"}
```

## 强制关闭占用进程

如果确定可以关闭：

```bash
# 关闭 PHP 进程
taskkill /F /IM php.exe

# 关闭 Node 进程
taskkill /F /IM node.exe
```

## 重启电脑后上传

如果以上方法都不行，最简单的方式：

1. **保存所有工作**
2. **重启电脑**
3. **不要打开任何编辑器**
4. **直接运行命令**：
   ```bash
   cd e:\123.207.52.46_8088\new_system
   git add .
   git commit -m "feat: 91家教管理系统初始版本"
   git push -u origin main
   ```

## 验证上传成功

上传完成后，访问：
```
https://gitee.com/yuchendeloy/tjiajiao91
```

检查文件是否都已上传。

## 注意事项

1. **确保 .gitignore 生效**：
   - backend/.env 不应该被上传
   - backend/cert/wechat/*.pem 不应该被上传
   - node_modules/ 不应该被上传

2. **测试文件不需要上传**：
   - test_*.php
   - check_*.php
   - fix_*.php
   - add_dispatcher_field.php

3. **如果不小心上传了敏感文件**：
   ```bash
   git rm --cached backend/.env
   git commit -m "chore: 删除敏感文件"
   git push
   ```

**建议**：先关闭所有程序，重启电脑后再尝试上传，这是最可靠的方法。
