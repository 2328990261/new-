# 上线文档：微信支付支持多套配置（数据库变更）

适用范围：`new_system` 后台「支付配置」已升级为 **微信支付多套配置**（按 `scene` 区分，例如 `default` / `h5`），并在下单时记录 `wechat_payment_config_id` 以保证回调验签/退款使用正确配置。

> 说明：你的线上库表有 `fa_` 前缀，以下 SQL 已按 `fa_` 编写。

---

## 1. 上线前检查

### 1.1 确认表存在

```sql
SHOW TABLES LIKE 'fa_payment_config';
SHOW TABLES LIKE 'fa_payments';
```

### 1.2 查看字段是否已存在（避免重复执行报错）

```sql
SHOW COLUMNS FROM fa_payment_config;
SHOW COLUMNS FROM fa_payments;
```

需要新增/确认的字段：
- `fa_payment_config`：`scene`、`name`、`is_default`
- `fa_payments`：`wechat_payment_config_id`

---

## 2. DDL 变更（加字段）

> 如果某个字段已存在，就跳过该条 SQL（否则会报 Duplicate column）。

### 2.0 `fa_payment_config` 增加 `refund_follow_qrcode`（如不存在）

> 你当前页面报错 `Unknown column 'refund_follow_qrcode'` 就是因为线上/本地表缺这个字段。

```sql
ALTER TABLE fa_payment_config
  ADD COLUMN refund_follow_qrcode varchar(255) NOT NULL DEFAULT '' COMMENT '退费页关注二维码' AFTER notify_url;
```

### 2.1 `fa_payment_config` 增加 `scene`

```sql
ALTER TABLE fa_payment_config
  ADD COLUMN scene varchar(32) NOT NULL DEFAULT 'default' COMMENT '配置场景(default/h5/...)' AFTER payment_method;
```

### 2.2 `fa_payment_config` 增加 `name`

```sql
ALTER TABLE fa_payment_config
  ADD COLUMN name varchar(64) NOT NULL DEFAULT '' COMMENT '配置名称' AFTER scene;
```

### 2.3 `fa_payment_config` 增加 `is_default`

```sql
ALTER TABLE fa_payment_config
  ADD COLUMN is_default tinyint(1) NOT NULL DEFAULT 0 COMMENT '是否默认(同method+scene唯一)' AFTER is_enabled;
```

### 2.4 `fa_payments` 增加 `wechat_payment_config_id`

```sql
ALTER TABLE fa_payments
  ADD COLUMN wechat_payment_config_id int(11) DEFAULT NULL COMMENT '微信支付配置ID' AFTER payment_method;
```

---

## 3. 索引变更（关键：允许同一 payment_method 多条）

### 3.1 删除 `payment_method` 单列唯一索引（如果存在）

如果你当前表结构里 `payment_method` 是 **unique**，会导致 `wechat` 只能存在一条，从而无法新增多套配置。

```sql
ALTER TABLE fa_payment_config DROP INDEX payment_method;
```

> 注意：如果索引名不是 `payment_method`，请在数据库工具里先看索引名，再替换上述 SQL 中的索引名。

### 3.2 增加联合唯一（推荐）

推荐加一个联合唯一，避免同一场景下名称重复（可按你业务习惯调整）。

```sql
ALTER TABLE fa_payment_config
  ADD UNIQUE KEY uk_method_scene_name (payment_method, scene, name);
```

---

## 4. 数据迁移（保证“原来那一套”可见且为默认）

### 4.1 为历史 wechat 记录补齐 `scene/name/is_default`

将最早的一条 `wechat + default` 设置为默认（MySQL 通用写法，避免 `UPDATE ... LIMIT` 兼容问题）。

```sql
UPDATE fa_payment_config
SET scene='default',
    name=IF(name='', '默认微信支付', name),
    is_default=1
WHERE id = (
  SELECT t.id FROM (
    SELECT id
    FROM fa_payment_config
    WHERE payment_method='wechat'
    ORDER BY id ASC
    LIMIT 1
  ) t
);
```

## 5. 上线后验证

### 5.1 验证表结构

```sql
SHOW COLUMNS FROM fa_payment_config LIKE 'scene';
SHOW COLUMNS FROM fa_payment_config LIKE 'name';
SHOW COLUMNS FROM fa_payment_config LIKE 'is_default';
SHOW COLUMNS FROM fa_payments LIKE 'wechat_payment_config_id';
```

### 5.2 验证索引

```sql
SHOW INDEX FROM fa_payment_config;
```

确认不再存在 `payment_method` 的单列 unique，且存在 `uk_method_scene_name`（或你定义的联合唯一）。

### 5.3 验证数据

```sql
SELECT id, payment_method, scene, name, is_enabled, is_default
FROM fa_payment_config
ORDER BY id DESC;
```

至少应看到：
- 一条 `payment_method=wechat AND scene=default AND is_default=1`（原来那一套）
- 你后续可新增多条 `wechat`（比如 `scene=h5`）

---

## 6. 回滚策略（仅数据库层面）

通常不建议直接删字段（会影响新代码），更推荐“保留字段 + 恢复索引/数据约束”。

### 6.1 如需暂时禁止多套（不推荐）

可以把联合唯一删掉，再加回单列 unique（会导致已有多条 wechat 无法保持，需要先清理数据）。

```sql
-- 删除联合唯一（按你的索引名）
ALTER TABLE fa_payment_config DROP INDEX uk_method_scene_name;

-- 加回单列 unique（仅在确保 payment_method 不重复时执行）
ALTER TABLE fa_payment_config ADD UNIQUE KEY payment_method (payment_method);
```

---

## 7. 业务配置建议

- `scene=default`：用于现有非 H5 场景（PC Native / 微信内 JSAPI 也可以仍用 default，按你后续页面决定）。
- `scene=h5`：用于你后续新做的 H5 支付/退费页面。
- 每个 `scene` 建议至少保持 **1 条启用且默认** 的配置。
