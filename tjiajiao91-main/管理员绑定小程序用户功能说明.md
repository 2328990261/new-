# 管理员绑定小程序用户功能说明

## 📋 功能概述

为管理员表（`fa_admin`）添加 `openid` 字段，实现管理员与小程序用户（`fa_users`）的绑定关系。

## 🗄️ 数据库结构

### 现有表结构

**fa_admin 表**（管理员表）
```sql
- id (主键)
- nickname (昵称)
- role (角色：super_admin/customer_service/team_leader/dispatcher)
- leader_id (组长ID)
- status (状态)
- contact (联系方式)
- wechat_qrcode (微信二维码)
- email (邮箱)
- last_login_time (最后登录时间)
- username (账号)
- password (密码)
- create_time (创建时间)
- update_time (更新时间)
```

**fa_users 表**（小程序用户表）
```sql
- id (主键)
- openid (微信openid，唯一)
- phone (手机号)
- nickname (昵称)
- avatar (头像)
- create_time (创建时间)
- update_time (更新时间)
- platform (平台：miniprogram/h5)
```

### 新增字段

在 `fa_admin` 表中新增：
- `openid` VARCHAR(100) - 绑定的小程序用户openid（唯一索引）
- `bind_time` DATETIME - 绑定时间

## 🔌 API 接口说明

### 1. 绑定小程序用户

**接口**：`POST /admin/api/admin-manage/bindMiniUser/{id}`

**参数**：
```json
{
  "openid": "opYqv7ULMDsSjxZ7bwRvYAeNTRo8"
}
```

**说明**：
- `{id}` 是管理员ID
- `openid` 是小程序用户的openid（从 fa_users 表获取）

**成功响应**：
```json
{
  "success": true,
  "message": "绑定成功",
  "data": {
    "openid": "opYqv7ULMDsSjxZ7bwRvYAeNTRo8",
    "bind_time": "2026-03-09 12:00:00",
    "user_info": {
      "nickname": "宇晨Eugene",
      "phone": "15986845083",
      "avatar": "http://tmp/..."
    }
  }
}
```

**错误响应**：
```json
{
  "success": false,
  "error": "该小程序用户已被其他管理员绑定"
}
```

### 2. 解绑小程序用户

**接口**：`POST /admin/api/admin-manage/unbindMiniUser/{id}`

**参数**：无

**成功响应**：
```json
{
  "success": true,
  "message": "解绑成功"
}
```

### 3. 查询绑定信息

**接口**：`GET /admin/api/admin-manage/getMiniUserInfo/{id}`

**参数**：无

**成功响应**：
```json
{
  "success": true,
  "data": {
    "openid": "opYqv7ULMDsSjxZ7bwRvYAeNTRo8",
    "bind_time": "2026-03-09 12:00:00",
    "user_info": {
      "id": 1,
      "nickname": "宇晨Eugene",
      "phone": "15986845083",
      "avatar": "http://tmp/...",
      "platform": "miniprogram",
      "create_time": "2025-12-09 14:23:19"
    }
  }
}
```

## 🎯 业务规则

1. **唯一性约束**
   - 一个管理员只能绑定一个小程序用户
   - 一个小程序用户只能被一个管理员绑定
   - openid 有唯一索引保证数据完整性

2. **验证规则**
   - 绑定前验证 openid 是否存在于 fa_users 表
   - 绑定前检查该 openid 是否已被其他管理员绑定
   - 解绑前检查管理员是否已绑定

3. **自动记录**
   - 绑定时自动记录 bind_time
   - 解绑时清空 openid 和 bind_time
