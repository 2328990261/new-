# 线索管理 - 客服组长分配功能修改说明

## 修改日期
2026-03-11

## 修改内容

### 1. 功能需求
允许客服组长（team_leader）在线索管理中分配订单给自己组内的客服成员。客服通过 `fa_admin` 表的 `leader_id` 字段绑定到客服组长。

### 2. 数据库结构
- **表名**: `fa_admin`
- **关键字段**:
  - `id`: 管理员ID
  - `role`: 角色（super_admin/team_leader/customer_service/dispatcher）
  - `leader_id`: 组长ID，关联admin表，用于客服绑定到组长
  - `status`: 状态（0-禁用，1-启用）

### 3. 修改的文件

#### 3.1 Lead.php - 批量分配方法 (batchAssign)
**文件路径**: `tjiajiao91-main/new_system/backend/app/controller/admin/Lead.php`

**修改内容**:
- 增加了组长分配权限验证
- 当角色为 `team_leader` 时，验证被分配的客服是否属于自己的组
- 只允许分配给通过 `leader_id` 绑定的组员（包含组长自己）

**关键逻辑**:
```php
// 如果是组长，验证被分配的客服是否属于自己的组
if ($adminRole === 'team_leader') {
    // 获取组员ID列表（包含自己）
    $teamMemberIds = Admin::where('leader_id', $adminId)
        ->where('status', '=', 1)
        ->column('id');
    $teamMemberIds[] = $adminId; // 包含自己
    
    // 验证被分配的客服是否在组内
    if (!in_array($assignedAdminId, $teamMemberIds)) {
        return json(['success' => false, 'error' => '只能分配给本组成员']);
    }
}
```

#### 3.2 Lead.php - 更新线索方法 (update)
**文件路径**: `tjiajiao91-main/new_system/backend/app/controller/admin/Lead.php`

**修改内容**:
- 增加了组长编辑权限验证
- 组长可以编辑分配给组员的线索
- 组长重新指派客服时，验证新客服是否在组内

**关键逻辑**:
```php
// 权限验证逻辑：
// 1. 超级管理员可以编辑所有线索
// 2. 分配给自己的线索可以编辑
// 3. 未分配的线索（公共池）可以编辑
// 4. 组长可以编辑分配给组员的线索
$hasPermission = false;

if ($adminRole === 'super_admin') {
    $hasPermission = true;
} elseif ($lead->assigned_admin_id == $adminId) {
    $hasPermission = true;
} elseif ($lead->assigned_admin_id == 0) {
    $hasPermission = true;
} elseif ($adminRole === 'team_leader') {
    // 组长可以编辑组员的线索
    $teamMemberIds = Admin::where('leader_id', $adminId)
        ->where('status', '=', 1)
        ->column('id');
    if (in_array($lead->assigned_admin_id, $teamMemberIds)) {
        $hasPermission = true;
    }
}
```

#### 3.3 AdminManage.php - 获取客服列表方法 (getAllCustomerServices)
**文件路径**: `tjiajiao91-main/new_system/backend/app/controller/admin/AdminManage.php`

**修改内容**:
- 当组长获取客服列表时，只返回自己组内的客服
- 通过 `leader_id` 字段过滤，只显示绑定到该组长的客服（包含组长自己）

**关键逻辑**:
```php
// 如果是组长，只能看到自己组内的客服（包含自己）
if ($adminRole === 'team_leader') {
    $teamMemberIds = Admin::where('leader_id', $adminId)
        ->where('status', '=', 1)
        ->column('id');
    $teamMemberIds[] = $adminId; // 包含自己
    $query->whereIn('id', $teamMemberIds);
}
```

### 4. 权限说明

| 角色 | 查看权限 | 编辑权限 | 分配权限 |
|------|---------|---------|---------|
| super_admin（超级管理员） | 所有线索 | 所有线索 | 可以分配给任何客服 |
| team_leader（客服组长） | 团队线索（自己+组员） | 自己的线索、组员的线索、公共池线索 | 只能分配给本组成员（通过leader_id绑定） |
| customer_service（客服） | 自己的线索 | 自己的线索、公共池线索 | 无分配权限 |

### 5. 使用场景

1. **绑定客服到组长**:
   - 在管理员管理中，设置客服的 `leader_id` 为组长的 `id`
   - 例如：客服ID=29的 `leader_id=18`，表示该客服属于ID=18的组长

2. **组长分配线索**:
   - 组长登录后，在线索管理中选择线索
   - 点击"分配"按钮，只能看到自己组内的客服列表
   - 选择组员后完成分配

3. **组长编辑线索**:
   - 组长可以编辑分配给自己的线索
   - 组长可以编辑分配给组员的线索
   - 组长可以在编辑时重新指派客服（仅限本组成员）

4. **权限验证**:
   - 如果组长尝试分配给非本组成员，系统会提示"只能分配给本组成员"
   - 如果组长尝试编辑非本组成员的线索，系统会提示"无权限编辑此线索"
   - 如果组长在编辑时重新指派给非本组成员，系统会提示"只能指派给本组成员"
