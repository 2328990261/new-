## 2. 开发环境搭建（首次）

### 2.1 检查清单

- [ ] 安装 PHP 7.4+、Composer、MySQL、Node.js 18+
- [ ] 克隆仓库并切换到 `91家教最新版` 分支
- [ ] 后端：`composer install`，复制 `.env.example` → `.env`
- [ ] 配置 MySQL 数据库并导入基础表结构
- [ ] Web 服务器根目录指向 `backend/public/`
- [ ] 管理端：`npm install && npm run dev`
- [ ] 用户站：`npm install && npm run dev`
- [ ] 访问 `/api/cities` 验证后端
- [ ] 分别登录管理端与用户站验证前端代理

### 2.2 推荐本地端口

| 服务 | 默认/常见端口 | 配置位置 |
|------|---------------|----------|
| 后端 PHP | 8000 或 80 | phpStudy / 内置服务器 |
| 管理端 Vite | 5173 | `frontend/admin/vite.config.*` |
| 用户站 Vite | 3001 或 5174 | `frontend/user/vite.config.*` |

后端地址需与 `.env.development` 中 `VITE_BACKEND_URL` 一致。

### 2.3 后端本地启动示例

```bash
cd tjiajiao91-main/new_system/backend
php think run -p 8000
```

或使用 phpStudy 将站点根目录设为 `backend/public`。

## 3. 日常开发流程

### 3.1 分支策略

| 分支 | 用途 |
|------|------|
| `91家教最新版` | 当前一体化开发快照（推荐） |
| `main` | 历史主分支 |
| `feature/*` | 功能开发 |
| `backup/*` | 备份快照 |

新功能建议从 `91家教最新版` 拉取 feature 分支，完成后合并回主开发分支。

### 3.2 提交规范

```
feat:     新功能
fix:      修复 Bug
docs:     文档更新
style:    格式调整（不影响逻辑）
refactor: 重构
test:     测试相关
chore:    构建/配置/脚本
```

示例：

```bash
git add .
git commit -m "fix: 修复教师注册页面上传 token 读取方式"
```

### 3.3 代码修改常见位置

| 需求类型 | 后端 | 前端 |
|----------|------|------|
| 新增 API | `route/api.php` 或 `admin.php` + 控制器 | `src/api/*.js` |
| 数据表变更 | `sql/` 新建脚本 + Model | — |
| 管理页功能 | `controller/admin/` | `frontend/admin/src/views/admin/` |
| 用户站页面 | `controller/api/` | `frontend/user/src/views/` |
| 小程序接口 | `controller/api/` | 小程序 `utils/api.js` |
| 跨域/CORS | `middleware/Cors.php` | Vite proxy 配置 |
| 权限控制 | `middleware/Auth.php`、Admin 角色 | 路由守卫 |

### 3.4 修改配置后必做

| 变更 | 操作 |
|------|------|
| 修改 `.env.development` | 重启 Vite（Ctrl+C 后 `npm run dev`） |
| 修改后端 `.env` | 重启 PHP / 清除 `runtime/cache/` |
| 修改路由文件 | 清除 ThinkPHP 缓存 |
| 修改 SQL 结构 | 执行对应 `sql/` 脚本并更新 Model |

## 4. 联调与测试

### 4.1 后端 API 快速验证

```bash
# 城市列表（公开接口）
curl http://localhost:8000/api/cities

# 管理端登录
curl -X POST http://localhost:8000/admin/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"***"}'
```

### 4.2 前端联调

1. 打开浏览器开发者工具 → Network
2. 确认请求 URL 与状态码
3. 401/403：检查 Token、中间件、角色权限
4. 404：检查后端路由、`.htaccess`、VITE_BACKEND_URL（见 UPLOAD_FIX_GUIDE）
5. CORS 错误：检查 Cors 中间件与 Vite 代理

### 4.3 图片上传测试

1. 教师端登录后进入注册页
2. 上传头像，观察 Network 中 `POST /api/teachers/upload`
3. 失败时查看 `backend/runtime/log/` 日志（搜索 `Upload`）
4. 参考 [UPLOAD_FIX_GUIDE.md](../UPLOAD_FIX_GUIDE.md)

### 4.4 小程序联调

1. 配置 `config/env.js` 的 `baseUrl`
2. 开发阶段可在开发者工具关闭「域名校验」
3. 真机预览需 HTTPS 合法域名
4. 微信/支付宝 AppID 与后端 `.env` 配置一致

## 5. 数据库运维

### 5.1 执行 SQL 脚本

**Windows：**

```cmd
cd tjiajiao91-main\new_system\backend\sql
deploy_teacher_verification_codes.bat
```

**Linux/Mac：**

```bash
cd tjiajiao91-main/new_system/backend/sql
chmod +x deploy_teacher_verification_codes.sh
./deploy_teacher_verification_codes.sh
```

脚本会从 `backend/.env` 读取数据库连接。

### 5.2 常用表前缀

业务表统一使用 **`fa_`** 前缀，例如：

- `fa_teacher_verification_codes` — 教师验证码
- `fa_personnel_salary` — 人事薪酬
- `fa_admin` — 管理员

## 6. 构建与发布

### 6.1 前端生产构建

```bash
# 管理端
cd tjiajiao91-main/new_system/frontend/admin
npm run build
# 产物：dist/

# 用户站
cd tjiajiao91-main/new_system/frontend/user
npm run build
# 产物：dist/
```

生产环境变量使用 `.env.production`，部署前核对 `VITE_BACKEND_URL` 与 API 路径。

### 6.2 后端上线

1. `composer install --no-dev --optimize-autoloader`
2. 上传代码（排除 `runtime/` 缓存，保留目录结构）
3. 配置 `.env`（服务器本地，不入库）
4. 设置 `public/` 为 Web 根目录
5. `public/uploads/` 及子目录可写
6. 清除 `runtime/cache/`

### 6.3 小程序发布

1. HBuilderX：发行 → 小程序-微信/支付宝
2. 开发者工具上传版本
3. 公众平台提交审核 → 发布
4. 同步更新后端小程序配置（AppID、密钥等）

## 7. 日志与排错

### 7.1 日志位置

| 类型 | 路径 |
|------|------|
| 后端应用日志 | `backend/runtime/log/` |
| 前端控制台 | 浏览器 F12 → Console / Network |
| 小程序 | 开发者工具调试器 |
| PHP 错误 | phpStudy / 服务器 error_log |

### 7.2 常见问题速查

| 现象 | 排查步骤 |
|------|----------|
| API 404 | ① Web 根目录是否 `public/` ② `.htaccess` / Nginx 重写 ③ 路由文件 ④ 清缓存 |
| API 500 | ① `runtime/log/` ② PHP 扩展 ③ 目录权限 ④ `.env` 数据库 |
| 上传失败 | ① `uploads/` 权限 ② PHP `upload_max_filesize` ③ Token ④ CORS |
| 登录失败 | ① 验证码表是否存在 ② 密码/账号 ③ 中间件 Token |
| 小程序请求失败 | ① 合法域名 ② baseUrl ③ HTTPS ④ 后端微信配置 |
| 支付回调异常 | ① `PAYMENT_NOTIFY_URL` ② HTTPS ③ 商户平台配置 |

### 7.3 清除 ThinkPHP 缓存

```bash
# 删除缓存目录内容（保留目录）
rm -rf tjiajiao91-main/new_system/backend/runtime/cache/*
```

Windows 可手动删除 `runtime\cache\` 下文件。

## 8. 安全规范

1. **禁止** 将 `.env`、密钥、AppSecret 提交到 Git
2. 生产环境关闭 ThinkPHP 调试模式
3. 管理端使用强密码，定期更换
4. 上传目录禁止执行 PHP（Nginx/Apache 配置）
5. API 敏感操作需认证中间件校验
6. 支付、退款相关变更需双人复核
