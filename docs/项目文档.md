## 1. 项目概述

**91家教（tjiajiao91）** 是一套面向家教中介/平台的综合业务系统，覆盖：

- **管理后台**：订单、教师、线索、支付、企业人事、小程序配置等
- **用户 Web 站**：教师注册/登录、家教信息浏览、预约、支付与退款
- **微信小程序**：家长预约、教师库、AI 预约、订单与个人中心
- **支付宝小程序**：与微信端功能对齐的多端适配

系统采用 **前后端分离** 架构：后端 ThinkPHP 6 提供 REST API，前端 Vue 3 + Vite，小程序基于 uni-app。

## 6. 快速部署

### 6.1 克隆代码

```bash
git clone https://github.com/2328990261/new-.git
cd new-
git checkout "91家教最新版"   # 若存在该分支
```

### 6.2 后端

```bash
cd tjiajiao91-main/new_system/backend
composer install
cp .env.example .env
# 编辑 .env，配置数据库、微信/支付宝、邮件等
```

**Web 根目录** 指向 `backend/public/`，并启用 URL 重写（Apache `mod_rewrite` 或 Nginx `try_files`）。

验证：浏览器访问 `http://localhost/api/cities`，应返回 JSON 城市列表。

### 6.3 管理端前端

```bash
cd tjiajiao91-main/new_system/frontend/admin
npm install
npm run dev
```

默认开发端口见 Vite 输出；`.env.development` 中 `VITE_BACKEND_URL` 指向后端地址。

### 6.4 用户站前端

```bash
cd tjiajiao91-main/new_system/frontend/user
npm install
npm run dev
```

### 6.5 数据库脚本

脚本位于 `tjiajiao91-main/new_system/backend/sql/`：

| 脚本/文档 | 用途 |
|-----------|------|
| `create_teacher_verification_codes.sql` | 教师验证码表 |
| `README_TEACHER_VERIFICATION.md` | 验证码表部署说明 |
| `deploy_teacher_verification_codes.bat/.sh` | 一键导入 |
| `create_personnel_salary_table.sql` | 人事薪酬表 |
| `README_PERSONNEL_SALARY.md` | 薪酬模块说明 |
| `deploy_personnel_salary.bat/.sh` | 薪酬表部署 |

执行前请备份数据库，并确认 `.env` 中数据库连接正确。

## 7. 配置说明

### 7.1 后端 `.env` 关键项

| 变量 | 说明 |
|------|------|
| `DATABASE_*` | MySQL 连接 |
| `PAYMENT_NOTIFY_URL` | 微信支付异步通知 URL |
| `COZE_RESUME_*` | Coze 简历智能分析（教师端） |
| `ALIPAY_PHONE_AES_KEY` | 支付宝手机号解密 |
| `TENCENT_MAP_KEY` | 腾讯地图逆地理编码 |
| `OFFICIAL_TOKEN_*` | 内部 token 接口白名单与签名 |

完整示例见 `backend/.env.example`。**切勿将真实 `.env` 提交到 Git。**

### 7.2 前端环境变量

**用户站**（`frontend/user/.env.development`）：

```env
VITE_API_BASE_URL=/api
VITE_BACKEND_URL=http://localhost:8000
```

**管理端**（`frontend/admin/.env.development`）：

```env
VITE_API_BASE_URL=/admin
VITE_BACKEND_URL=http://127.0.0.1:8000
```

修改 `.env` 后需重启 Vite 开发服务器。

### 7.3 小程序 API 地址

编辑各小程序目录下 `config/env.js` 中的 `baseUrl`，指向线上或本地后端 API 根地址。

## 8. API 路由概览

### 8.1 用户端（`/api/*`）

- 家教：`GET /api/tutor/list`、`/api/tutor/detail/:id`
- 教师：`POST /api/teachers/register`、`POST /api/teachers/upload`
- 教师注册：`POST /api/teacher-register/submit`
- 订单：`POST /api/order/booking`
- 微信：`POST /api/wechat/login`
- 支付：`POST /api/payment/*`
- 公共：`GET /api/cities`、`/api/subjects`

完整路由见 `backend/route/api.php`。

### 8.2 管理端（`/admin/api/*`）

- 认证：`POST /admin/api/login`
- 上传：`POST /admin/api/upload/image`
- 仪表盘、订单、教师、线索、支付、人事等

完整路由见 `backend/route/admin.php`。

## 11. 生产部署要点

1. **后端**：`public/` 为唯一 Web 入口；`runtime/`、`public/uploads/` 需可写
2. **前端**：`npm run build` 后将 `dist/` 部署到静态服务器或 CDN
3. **HTTPS**：支付、微信回调必须 HTTPS
4. **域名**：小程序后台配置 request 合法域名
5. **PHP 限制**：`upload_max_filesize`、`post_max_size` ≥ 5M
6. **缓存**：部署后清除 `backend/runtime/cache/`
