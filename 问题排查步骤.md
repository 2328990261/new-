# 组长编辑权限问题排查步骤

## 问题描述
客服组长只能编辑"已发单"状态的线索，其他状态都提示"无权限编辑此线索"。

## 排查步骤

### 步骤1: 验证数据库关系
在MySQL中执行以下SQL：

```sql
-- 查看组长和组员的关系
SELECT 
    id, 
    nickname, 
    username, 
    role, 
    leader_id, 
    status 
FROM fa_admin 
WHERE id IN (18, 29) OR leader_id = 18;
```

**预期结果**:
- ID 18: role='team_leader', leader_id=NULL, status=1
- ID 29: role='customer_service', leader_id=18, status=1

**如果不符合预期**，执行以下SQL修复：

```sql
-- 确保ID 29的leader_id指向18
UPDATE fa_admin SET leader_id = 18, status = 1 WHERE id = 29;

-- 确认修改
SELECT id, nickname, role, leader_id, status FROM fa_admin WHERE id IN (18, 29);
```

```sql

    (SELECT nickname FROM fa_admin WHERE id = fa_leads.assigned_admin_id) as assigned_to
```

## 常见问题

### Q1: 为什么只有"已发单"状态可以编辑？
A: 这不应该发生。权限验证不应该依赖线索状态。可能的原因：
1. 后端代码没有重启生效
2. 浏览器缓存了旧的代码
3. 有其他地方也在做权限验证

### Q2: 如何确认后端代码已经更新？
A: 在 `Lead.php` 的 `update` 方法开头添加：
```php
trace('UPDATE方法被调用 - 时间: ' . date('Y-m-d H:i:s'), 'info');
```
然后重启服务，尝试编辑，查看日志文件是否有这条记录。

### Q3: 如何确认Session中的角色信息正确？
A: 在浏览器控制台执行：
```javascript
// 查看当前请求的响应头
fetch('/admin/lead/index?view_scope=team')
  .then(r => r.json())
  .then(d => console.log(d))
```
