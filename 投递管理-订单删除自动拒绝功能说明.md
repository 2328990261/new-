# 投递管理 - 订单删除自动拒绝功能说明

## 问题描述
在管理端的投递管理中，当家教订单（单子）被删除时，系统之前**没有**自动将相关的投递状态设置为"未通过（已拒绝）"。

## 问题分析

### 原有逻辑
1. **家教订单删除**：在 `Tutor.php` 控制器中，删除订单时只是简单的物理删除，没有处理关联的投递记录
2. **投递记录关联**：投递记录（`resume_application` 表）通过 `tutor_id` 字段关联到家教订单（`tutor_orders_new` 表）
3. **孤儿数据问题**：当订单被删除后，投递记录会变成"孤儿数据"，状态仍然是"待审核"或"已通过"，但对应的订单已不存在

## 解决方案

### 修改内容
修改了 `tjiajiao91-main/new_system/backend/app/controller/admin/Tutor.php` 文件中的两个方法：

#### 1. 单个删除方法 `delete()`
在删除订单前，自动将关联的投递记录状态设置为"已拒绝"：

```php
// 在删除订单前，先将关联的投递记录状态设置为"已拒绝"
$adminId = $_SESSION['admin_id'];
$affectedApplications = \think\facade\Db::name('resume_application')
    ->where('tutor_id', $id)
    ->whereIn('status', ['pending', 'approved']) // 只处理待审核和已通过的投递
    ->update([
        'status' => 'rejected',
        'admin_remark' => '该家教订单已被删除',
        'review_time' => date('Y-m-d H:i:s'),
        'reviewer_id' => $adminId
    ]);
```

#### 2. 批量删除方法 `batchDelete()`
同样在批量删除订单前，自动将关联的投递记录状态设置为"已拒绝"：

```php
// 在删除订单前，先将关联的投递记录状态设置为"已拒绝"
$adminId = $_SESSION['admin_id'];
$affectedApplications = \think\facade\Db::name('resume_application')
    ->whereIn('tutor_id', $existingIds)
    ->whereIn('status', ['pending', 'approved']) // 只处理待审核和已通过的投递
    ->update([
        'status' => 'rejected',
        'admin_remark' => '该家教订单已被删除',
        'review_time' => date('Y-m-d H:i:s'),
        'reviewer_id' => $adminId
    ]);
```

### 功能特点

1. **自动处理**：删除订单时自动处理关联的投递记录，无需手动操作
2. **状态筛选**：只处理"待审核"（pending）和"已通过"（approved）状态的投递，已拒绝的不重复处理
3. **记录审核信息**：
   - 状态改为 `rejected`（已拒绝）
   - 备注说明 `该家教订单已被删除`
   - 记录审核时间和审核人ID
4. **日志记录**：在日志中记录影响的投递数量，便于追踪
5. **支持批量**：单个删除和批量删除都支持此功能

## 使用场景

### 场景1：删除单个订单
管理员在家教信息管理中删除某个订单时：
- 系统自动查找该订单的所有待审核/已通过的投递
- 将这些投递状态改为"已拒绝"
- 在投递记录中添加备注"该家教订单已被删除"

### 场景2：批量删除订单
管理员批量删除多个订单时：
- 系统自动查找这些订单的所有待审核/已通过的投递
- 批量将这些投递状态改为"已拒绝"
- 在投递记录中添加备注"该家教订单已被删除"

## 数据一致性保证

1. **事务处理**：删除操作在数据库事务中执行，确保数据一致性
2. **错误处理**：如果删除失败，投递状态也不会被修改
3. **日志追踪**：所有操作都有详细的日志记录，便于问题排查

## 测试建议

1. **单个删除测试**：
   - 创建一个家教订单
   - 让教师投递该订单（状态为"待审核"）
   - 删除该订单
   - 检查投递记录状态是否变为"已拒绝"，备注是否正确

2. **批量删除测试**：
   - 创建多个家教订单
   - 让教师投递这些订单
   - 批量删除这些订单
   - 检查所有投递记录状态是否都变为"已拒绝"

3. **边界情况测试**：
   - 删除没有投递记录的订单（应正常删除）
   - 删除只有"已拒绝"投递的订单（不应重复处理）
   - 删除有多个投递的订单（应全部处理）

## 注意事项

1. 此功能只处理"待审核"和"已通过"状态的投递，已经是"已拒绝"状态的投递不会重复处理
2. 删除操作是物理删除，订单数据会从数据库中永久删除，但投递记录会保留（状态改为已拒绝）
3. 审核人ID会记录为执行删除操作的管理员ID
4. 备注信息统一为"该家教订单已被删除"，便于识别
