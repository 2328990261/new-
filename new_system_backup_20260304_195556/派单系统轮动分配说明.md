# 派单系统轮动分配说明

## 功能概述
派单系统按照录入的家教单进行轮动分配，自动派给状态开启的派单组成员。

## 核心功能

### 1. 自动派单机制
- **触发时机**：创建家教订单时自动触发
- **分配策略**：基于工作量平衡的智能分配
- **派单对象**：状态为启用（status=1）的派单组成员

### 2. 轮动分配算法
```php
// 获取所有启用的派单组成员
$dispatchers = Admin::where('role', 'dispatcher')
    ->where('status', 1)
    ->order('id', 'asc')
    ->select();

// 计算每个派单员的工作量（今日已分配订单数）
foreach ($dispatchers as $dispatcher) {
    $workload = TutorOrder::where('dispatcher_id', $dispatcher['id'])
        ->where('status', 1)
        ->where('assigned_time', '>=', date('Y-m-d 00:00:00'))
        ->count();
}

// 选择工作量最少的派单员
usort($dispatcherWorkloads, function($a, $b) {
    return $a['workload'] - $b['workload'];
});
```

### 3. 派单字段说明
- `dispatcher_id`：派单员ID
- `contact_info`：派单员联系方式
- `assigned_time`：派单时间

### 2. 控制器实现

#### 自动派单方法 (`Tutor.php`)
```php
private function autoAssignOrder($order)
{
    // 获取启用的派单组成员
    $dispatchers = Admin::where('role', 'dispatcher')
        ->where('status', 1)
        ->order('id', 'asc')
        ->select();
    
    // 计算工作量并选择最优派单员
    $selectedDispatcher = $this->selectOptimalDispatcher($dispatchers);
    
    // 更新订单派单信息
    $order->dispatcher_id = $selectedDispatcher['id'];
    $order->contact_info = $selectedDispatcher['contact'];
    $order->assigned_time = date('Y-m-d H:i:s');
    $order->save();
}
```

#### 派单管理控制器 (`OrderAssign.php`)
- `pending()`：获取待派单列表
- `assigned()`：获取已派单列表
- `batchAssign()`：批量派单
- `assign()`：单个订单派单
- `cancel()`：取消派单
- `statistics()`：派单统计

## 使用流程

### 1. 设置派单组成员
1. 在管理员管理中创建派单员账号
2. 设置角色为 `dispatcher`
3. 确保状态为启用（`status=1`）
4. 填写联系方式（`contact`字段）

### 2. 自动派单流程
1. 管理员创建家教订单
2. 系统自动计算各派单员工作量
3. 选择工作量最少的派单员
4. 更新订单派单信息
5. 记录派单日志

### 3. 手动派单管理
1. 查看待派单列表
2. 批量选择订单进行派单
3. 或单独指定派单员
4. 支持取消派单重新分配

## 派单策略

### 1. 工作量平衡
- 优先分配给今日工作量最少的派单员
- 确保各派单员工作量相对均衡
- 避免某个派单员过载

### 2. 容错机制
- 没有派单员时不影响订单创建
- 派单失败只记录日志，不中断流程
- 支持手动重新派单

### 3. 统计监控
- 实时统计各派单员工作量
- 提供派单效率分析
- 支持派单历史查询

## 前端显示

### 1. 用户端显示
- 显示派单组昵称
- 提供联系方式复制功能
- 未派单时显示"暂无联系方式"

### 2. 管理端显示
- 派单状态筛选
- 派单员工作量统计
- 批量派单操作界面

## 注意事项

1. **派单员状态**：只有状态为启用的派单员才会参与自动分配
2. **工作量计算**：基于当日已分配订单数量计算工作量
3. **联系方式**：派单员的联系方式会自动填入订单的contact_info字段
4. **日志记录**：所有派单操作都会记录详细日志
5. **权限控制**：只有管理员可以查看和操作派单功能
