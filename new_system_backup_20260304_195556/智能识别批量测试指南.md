## ⚠️ 注意事项

### 文本分割规则
- ✅ **空行分割**：每个家教单之间需要有**空行**（至少一行空行）
- ❌ **单换行无效**：如果只是单个换行，不会分割

**正确示例**：
```
家教单 1 的内容...
家教单 1 的内容...
                    ← 空行
家教单 2 的内容...
家教单 2 的内容...
```

**错误示例**：
```
家教单 1 的内容...
家教单 1 的内容...
家教单 2 的内容...  ← 没有空行，会被识别为同一个家教单
家教单 2 的内容...
```

### 识别准确性

#### 提高识别率的技巧：
1. **城市+区域**：尽量同时包含城市和区域信息（如"深圳南山区"）
2. **年级+科目**：清晰标注年级和科目（如"初二数学"）
3. **薪资格式**：使用标准格式（如"150-180/次"、"200元/小时"）
4. **开头标识**：建议以"S"或其他标识开头，包含关键信息

#### 识别优先级：
1. 城市+区域 > 仅区域 > 仅城市
2. 精确匹配 > 模糊匹配
3. 数据库科目 > 正则匹配

## 🐛 问题排查

### 问题1: 没有分割成多个家教单
**症状**：输入了多个家教单，但只识别出1个

**解决方案**：
1. 检查家教单之间是否有**空行**
2. 在控制台查看 `📝 输入文本预览`，确认文本格式正确

### 问题2: 城市/区域/科目显示"未知"
**症状**：识别结果中名称显示为"未知"

**排查步骤**：
1. 查看控制台日志，找到对应的识别项
2. 检查是否有 `✅ 城市名称（后端已返回）` 日志
3. 如果没有，查看 `🔍 查找城市 ID` 的结果
4. 检查 ID 是否在数据库中存在

**解决方案**：
- 确保数据库中有对应的城市/区域/科目数据
- 检查状态字段是否为启用（status = 1）

### 问题3: 年级显示简写而不是完整名称
**症状**：显示"小六"而不是"小学六年级"

**解决方案**：
- 检查 `gradeMapping` 是否包含该年级的映射
- 查看控制台 `📚 年级映射` 日志
- 如果映射没生效，可能需要添加新的映射规则
