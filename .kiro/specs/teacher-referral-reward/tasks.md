# 实施计划：教师推荐奖励功能

## 概述

本实施计划将教师推荐奖励功能分解为可执行的开发任务。系统包含数据库设计、后端API、小程序端界面和管理后台四大模块。技术栈：PHP（ThinkPHP框架）+ MySQL + uni-app（Vue3）小程序。

## 注意事项

- 所有涉及推荐关系和抵扣券的操作必须使用数据库事务
- 抵扣券采用人工核销方式，不涉及支付流程，完全由管理员后台操作
- 抵扣券状态简化为：issued（已发放）→ verified（已核销）或 expired（已过期）
- 推荐码必须全局唯一，建议使用8位字母数字组合
- 排行榜数据需要缓存以提升性能
- 所有管理员操作需要记录审计日志
- 防作弊机制必须在推荐关系建立时执行
- 微信模板消息发送失败需要重试机制
