# 预约订单管理员关联功能实现说明（基于 OpenID）

## 功能概述
实现了管理员通过小程序分享预约表单给用户填写后，订单自动关联到该管理员的功能。基于管理员的 openid 进行关联，包括：
1. 管理员在小程序中登录后分享预约表单（链接包含管理员的 openid）
2. 用户提交预约后，通过 openid 找到对应的管理员并发送邮箱通知
3. 预约订单的备注中显示管理员昵称
4. 审核通过后创建的家教订单也关联到该管理员
5. 后台"我的订单"显示该管理员的昵称

## 实现步骤

### 1. 小程序端修改

#### 1.1 预约表单页面 (step-booking/index.vue)

**添加 adminOpenid 数据属性：**
```javascript
data() {
  return {
    safeAreaBottom: 0,
    adminOpenid: null, // 从URL获取的管理员openid
    currentStep: 0,
    // ...其他属性
  }
}
```

**在 onLoad 中获取URL参数：**
```javascript
onLoad(options) {
  const systemInfo = uni.getSystemInfoSync()
  this.safeAreaBottom = systemInfo.safeAreaInsets?.bottom || 0
  
  // 获取URL参数中的管理员openid
  if (options.admin_openid) {
    this.adminOpenid = options.admin_openid
    console.log('从URL获取管理员openid:', this.adminOpenid)
  }
  
  this.loadSubjects()
  this.loadUserInfo()
  this.loadSavedData()
}
```

**提交表单时传递 adminOpenid：**
```javascript
const res = await bookingApi.createBooking({
  user_id: userInfo.id,
  admin_openid: this.adminOpenid, // 传递管理员openid
  booking_data: bookingData
})
```

### 2. 后端接口修改

#### 2.1 小程序预约控制器 (MiniProgramBooking.php)

**通过 openid 查找管理员：**
```php
public function create()
{
    $userId = $this->request->post('user_id');
    $adminOpenid = $this->request->post('admin_openid'); // 接收管理员openid
    $bookingData = $this->request->post('booking_data');
    
    // ...验证代码
    
    // 通过openid查找管理员
    $admin = null;
    $adminId = 0;
    $adminNickname = null;
    if ($adminOpenid) {
        $admin = \app\model\Admin::where('openid', $adminOpenid)->find();
        if ($admin) {
            $adminId = $admin->id;
            $adminNickname = $admin->nickname ?: $admin->username;
        }
    }
    
    // 创建订单时保存管理员信息
    $order = ParentOrder::create([
        'order_no' => $orderNo,
        'admin_id' => $adminId,  // 保存管理员ID
        // ...其他字段
        'remark' => $adminNickname ? "小程序预约（{$adminNickname}）" : '小程序预约',  // 在备注中显示管理员昵称
        // ...
    ]);
    
    // 发送邮件通知给管理员
    if ($admin && $admin->email) {
        try {
            \app\service\EmailService::sendBookingNotification($admin, $order);
        } catch (\Exception $e) {
            // 邮件发送失败不影响订单创建
            trace('发送预约通知邮件失败: ' . $e->getMessage(), 'error');
        }
    }
}
```

#### 2.2 订单审核控制器 (Order.php)

**审核通过时填充 admin_openid：**
```php
// 创建家教信息（手动设置ID）
$tutorData = [
    'id' => $tutorId,
    'content' => $tutorContent,
    // ...其他字段
    'admin_id' => $order->admin_id ?: 0,
    'admin_openid' => null,  // 先设为null，后面根据admin_id填充
    // ...
];

// 如果预约订单有关联的管理员，填充 admin_openid
if ($order->admin_id > 0) {
    $orderAdmin = \app\model\Admin::find($order->admin_id);
    if ($orderAdmin && !empty($orderAdmin->openid)) {
        $tutorData['admin_openid'] = $orderAdmin->openid;
    }
}

$tutor = TutorOrder::create($tutorData);
```

## 使用方法

### 管理员分享预约链接

1. **管理员在小程序中登录**
   - 小程序会获取管理员的 openid

2. **生成分享链接：**
```javascript
// 在小程序中获取当前用户的openid
const userInfo = auth.getUserInfo()
const openid = userInfo.openid

// 生成分享链接
const shareUrl = `/pages/step-booking/index?admin_openid=${openid}`
```

3. **分享链接示例：**
```
/pages/step-booking/index?admin_openid=opYqv7ULMDsSjxZ7bwRvYAeNTRo8
```

4. **用户访问链接并填写表单**

5. **提交后的效果：**
   - 后端通过 openid 查找对应的管理员
   - 管理员收到邮箱通知（如果配置了邮箱）
   - 预约订单的备注显示：`小程序预约（管理员昵称）`
   - 订单关联到该管理员（admin_id 字段）

6. **审核通过后：**
   - 创建的家教订单也关联到该管理员
   - 家教订单的 `admin_openid` 字段自动填充
   - 小程序端可以通过 openid 查询"我的订单"

## 数据流程

```
管理员在小程序登录（获取openid）
    ↓
生成分享链接（包含openid）
    ↓
用户访问链接（带admin_openid参数）
    ↓
填写预约表单
    ↓
提交订单（包含admin_openid）
    ↓
后端通过openid查找管理员
    ├─ 保存 admin_id
    ├─ 备注中显示管理员昵称
    └─ 发送邮件通知给管理员
    ↓
管理员审核通过
    ↓
创建家教订单
    ├─ 继承预约订单的 admin_id
    └─ 自动填充 admin_openid
    ↓
小程序端可查询"我的订单"
```

## 显示效果

### 管理端预约订单列表
- **备注列**：显示 `小程序预约（张三）` 或 `小程序预约`
- **归属管理员列**：显示管理员昵称（通过 admin_id 关联）

### 管理端家教订单列表
- **下单渠道列**：显示 `小程序`
- **归属管理员列**：显示管理员昵称（通过 admin_id 关联）

### 小程序端
- 管理员可以通过 `/api/tutor/my-orders?openid=xxx` 查询自己的订单

## 关键优势

1. **基于 OpenID**：使用小程序的 openid 作为唯一标识，无需手动输入管理员ID
2. **自动关联**：通过 openid 自动查找并关联管理员
3. **邮件通知**：自动发送邮件通知给对应的管理员
4. **显示昵称**：在备注和列表中显示管理员昵称，方便识别
5. **完整追踪**：从预约到审核到家教订单，全程关联管理员

## 注意事项

1. **管理员必须绑定 openid**：管理员需要先在小程序中登录，系统才能获取其 openid
2. **邮件通知**：需要确保管理员配置了邮箱地址
3. **向后兼容**：如果没有传递 admin_openid，订单仍然可以正常创建（admin_id = 0）
4. **备注字段**：通过备注字段显示管理员昵称，方便在列表中识别
5. **openid 唯一性**：每个管理员的 openid 是唯一的，确保准确关联
