# 支付配置验证和测试功能说明

## 功能概述

支付配置页面现在包含完整的数据验证和真实连接测试功能，确保配置的正确性。

## 一、数据格式验证（保存时自动执行）

### 微信支付验证规则

| 字段 | 验证规则 | 示例 |
|------|---------|------|
| 应用ID | 必填，wx开头的18位字符 | wx1234567890abcdef |
| 商户号 | 必填，10位数字 | 1234567890 |
| API密钥 | 必填，32位字母数字 | abcd1234efgh5678ijkl9012mnop3456 |
| 应用密钥 | 可选，如填写必须32位字母数字 | - |
| 回调地址 | 可选，必须HTTPS协议 | https://yourdomain.com/notify |
| 证书路径 | 可选，必须.pem或.p12文件 | /path/to/cert.pem |

### 支付宝验证规则

| 字段 | 验证规则 | 示例 |
|------|---------|------|
| 应用ID | 必填，16位数字 | 2021001234567890 |
| 支付宝公钥 | 必填，至少200字符，Base64格式 | MIIBIjANBgkqhkiG9w0B... |
| 应用私钥 | 必填，至少200字符，Base64格式 | MIIEvQIBADANBgkqhki... |
| 回调地址 | 可选，必须http://或https:// | https://yourdomain.com/notify |

## 二、测试配置功能（手动触发）

### 微信支付测试

点击"测试配置"按钮后，系统会：

1. **自动保存当前配置**
2. **验证配置格式**
   - AppID格式
   - 商户号格式
   - API密钥长度
   - 回调地址格式
   - 证书文件存在性

3. **测试API连通性**
   - 调用微信支付订单查询接口
   - 使用测试订单号验证签名
   - 检查返回结果

4. **返回测试结果**
   - ✅ 成功：显示"配置验证通过，微信支付API连通正常"
   - ⚠️ 警告：配置格式正确但API连通失败（可能是网络问题）
   - ❌ 失败：显示具体的错误信息

### 支付宝测试

点击"测试配置"按钮后，系统会：

1. **自动保存当前配置**
2. **验证配置格式**
3. **测试API连通性**（待实现）
4. **返回测试结果**

## 三、使用流程

### 配置微信支付

1. **开启微信支付开关**
2. **填写必填信息**
   - 应用ID：从微信商户平台获取
   - 商户号：从微信商户平台获取
   - API密钥：在微信商户平台设置

3. **填写可选信息**
   - 应用密钥：用于JSAPI支付
   - 证书路径：用于退款等操作
   - 回调地址：支付成功后的通知地址

4. **点击"测试配置"**
   - 系统会自动保存并测试
   - 查看测试结果
   - 如果失败，根据错误提示修改

5. **点击"保存配置"**
   - 最终保存所有配置

### 配置支付宝

流程与微信支付类似。

## 四、错误提示示例

### 格式错误

```
微信支付：
- 应用ID格式不正确（应为wx开头的18位字符，如：wx1234567890abcdef）
- 商户号格式不正确（应为10位数字，如：1234567890）
- API密钥格式不正确（应为32位字符）
```

### API连通性错误

```
配置格式验证通过，但API连通性测试失败
警告：
- API连通性测试失败：通信失败：签名错误
- 回调地址建议使用HTTPS协议
```

### 成功提示

```
测试成功！配置验证通过，微信支付API连通正常
```

## 六、注意事项

1. **测试配置会自动保存**
   - 点击测试按钮会先保存配置
   - 无需先点保存再测试

2. **测试失败不影响保存**
   - 格式验证通过即可保存
   - API连通性失败可能是网络问题
   - 可以保存后稍后再测试

3. **生产环境建议**
   - 配置完成后务必测试
   - 确保回调地址可访问
   - 定期检查证书有效期

4. **安全建议**
   - API密钥不要泄露
   - 使用HTTPS回调地址
   - 定期更换密钥

## 七、常见问题

### Q: 测试失败但格式正确怎么办？
A: 可能是网络问题或微信服务器暂时不可用，可以稍后再试。格式正确的配置可以正常保存和使用。

### Q: 必须测试通过才能使用吗？
A: 不是。测试只是验证配置的正确性，即使测试失败，只要格式正确就可以保存。实际支付时会再次验证。

### Q: 回调地址必须配置吗？
A: 建议配置。回调地址用于接收支付成功的通知，不配置可能导致无法及时更新订单状态。

### Q: 证书文件必须配置吗？
A: 不是必须的。证书主要用于退款等操作，如果不需要退款功能可以不配置。
