# 快速修复支付页面404错误

## 错误信息
```
GET https://t.jiajiao91.com/api/api/agreement/payment 404 (Not Found)
```

## 快速修复步骤

### 1. 上传新文件

将以下文件上传到服务器对应位置：

**新增文件：**
```
new_system/backend/app/controller/api/Agreement.php
```
上传到：
```
/www/wwwroot/yourdomain.com/app/controller/api/Agreement.php
```

**更新文件：**
```
new_system/backend/route/api.php
```
上传到：
```
/www/wwwroot/yourdomain.com/route/api.php
```

### 2. 清除缓存

在宝塔面板的终端中执行：

```bash
cd /www/wwwroot/yourdomain.com
php think clear
```

或者在宝塔面板中：
1. 点击网站
2. 找到你的网站
3. 点击"设置"
4. 点击"PHP"
5. 点击"重载配置"

### 3. 测试接口

在浏览器中访问：
```
https://yourdomain.com/api/agreement/payment
```

应该看到类似这样的JSON响应：
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "id": 0,
    "type": "payment",
    "title": "91家教接单协议",
    "content": "<div>...</div>",
    "version": "1.0"
  }
}
```

### 4. 验证支付页面

访问支付页面：
```
https://yourdomain.com/payment
```

检查：
- ✅ 页面正常加载
- ✅ 没有404错误
- ✅ 协议弹窗可以打开
- ✅ 协议内容正确显示

## 如果还有问题

### 检查文件权限

```bash
chmod 644 /www/wwwroot/yourdomain.com/app/controller/api/Agreement.php
chmod 644 /www/wwwroot/yourdomain.com/route/api.php
```

### 检查PHP错误日志

在宝塔面板中：
1. 网站 → 设置 → 网站日志
2. 查看 error.log

### 重启PHP-FPM

在宝塔面板中：
1. 软件商店
2. 找到 PHP 7.x 或 8.x
3. 点击"重载配置"或"重启"

## 完成！

修复完成后，支付页面应该可以正常加载协议内容了。
