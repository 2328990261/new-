# 支付接口500错误 - 快速修复指南

## 错误信息
```
Call to undefined method app\service\WechatPayService::createH5Payment()
```

## 问题原因
生产环境的代码版本过旧，缺少支付方法。

## 快速修复步骤

### 方法1：运行编译脚本（推荐）

1. 双击运行根目录下的 `编译生产文件.bat`
2. 等待编译完成
3. 将 `deployment/backend/` 目录上传到服务器覆盖旧文件
4. 清除缓存：
   ```bash
   php think clear
   ```

### 方法2：手动复制文件

如果只想快速修复支付功能，只需复制以下文件：

```bash
# 1. 复制WechatPayService（包含所有支付方法）
copy new_system\backend\app\service\WechatPayService.php deployment\backend\app\service\

# 2. 复制Payment控制器（修复了字段名和异常处理）
copy new_system\backend\app\controller\api\Payment.php deployment\backend\app\controller\api\

# 3. 复制PaymentConfig模型（添加了异常处理）
copy new_system\backend\app\model\PaymentConfig.php deployment\backend\app\model\
```

然后上传到服务器并清除缓存。

### 方法3：直接在服务器上修复（临时方案）

如果无法立即更新代码，可以在服务器上临时修改：

编辑 `/www/wwwroot/jiajiaoxitong/backend/app/controller/api/Payment.php`

找到第261行附近的代码，将：
```php
$payResult = $wechatService->createH5Payment([...]);
```

改为：
```php
// 临时使用模拟支付
$payResult = $wechatService->createMockPayment([
    'order_no' => $orderNo,
    'amount' => $data['amount'],
    'body' => '家教信息服务费'
]);
```

## 验证修复

修复后，再次调用支付接口：
```bash
POST /api/payment/create
{
  "amount": 1,
  "payment_method": "wechat",
  "payer_name": "测试",
  "tutor_info": "1"
}
```

应该返回成功响应，包含二维码URL。

1. **启用真实支付**：在后台配置微信支付参数

3. **监控日志**：定期检查 `runtime/log/` 目录下的错误日志
