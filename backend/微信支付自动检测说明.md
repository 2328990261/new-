# 微信支付自动检测功能说明

## 功能概述

系统会自动检测用户的访问环境，并选择最合适的微信支付方式：

- **微信内** → JSAPI支付（公众号支付）
- **移动端浏览器** → H5支付
- **PC端浏览器** → Native扫码支付

## 自动检测逻辑

### 1. 检测微信环境
通过User-Agent判断是否包含 `MicroMessenger` 关键字：
- 是 → 使用 **JSAPI支付**
- 否 → 继续检测设备类型

### 2. 检测设备类型
通过User-Agent判断是否为移动设备：
- 移动设备 → 使用 **H5支付**
- PC设备 → 使用 **Native扫码支付**

### 3. 手动指定（可选）
前端可以通过参数强制指定支付类型：
```javascript
{
  trade_type: 'jsapi'  // 或 'h5' 或 'native'
}
```

## 支付方式详解

### JSAPI支付（微信内）
**使用场景：** 用户在微信公众号或微信内置浏览器中访问

**特点：**
- 需要用户的openid
- 支付体验最流畅
- 无需跳转，直接调起微信支付

**前端调用示例：**
```javascript
// 1. 先获取用户openid（通过微信授权）
// 2. 创建支付订单
const response = await axios.post('/api/payment/create', {
  amount: 100,
  payment_method: 'wechat',
  payer_name: '张三',
  openid: 'oxxxxxxxxxxxxxx'  // 必需
});

// 3. 调起微信支付
if (response.data.code === 200) {
  const { jsapi_params } = response.data.data;
  
  WeixinJSBridge.invoke('getBrandWCPayRequest', {
    appId: jsapi_params.appId,
    timeStamp: jsapi_params.timeStamp,
    nonceStr: jsapi_params.nonceStr,
    package: jsapi_params.package,
    signType: jsapi_params.signType,
    paySign: jsapi_params.paySign
  }, function(res) {
    if (res.err_msg === "get_brand_wcpay_request:ok") {
      // 支付成功
    }
  });
}
```

### H5支付（移动端浏览器）
**使用场景：** 用户在手机浏览器（非微信）中访问

**特点：**
- 会跳转到微信支付页面
- 支付完成后可返回原页面
- 适合移动端网页

**前端调用示例：**
```javascript
const response = await axios.post('/api/payment/create', {
  amount: 100,
  payment_method: 'wechat',
  payer_name: '张三',
  redirect_url: window.location.origin + '/payment-success'  // 支付完成后跳转
});

if (response.data.code === 200) {
  const { mweb_url } = response.data.data;
  // 跳转到微信支付页面
  window.location.href = mweb_url;
}
```

### Native扫码支付（PC端）
**使用场景：** 用户在PC浏览器中访问

**特点：**
- 生成二维码
- 用户用微信扫码支付
- 适合PC端网页

**前端调用示例：**
```javascript
const response = await axios.post('/api/payment/create', {
  amount: 100,
  payment_method: 'wechat',
  payer_name: '张三'
});

if (response.data.code === 200) {
  const { qrcode_url, code_url } = response.data.data;
  
  // 方式1：显示二维码图片
  document.getElementById('qrcode').src = qrcode_url;
  
  // 方式2：使用qrcode.js生成二维码
  new QRCode(document.getElementById('qrcode'), {
    text: code_url,
    width: 300,
    height: 300
  });
  
  // 轮询查询支付状态
  const timer = setInterval(async () => {
    const statusRes = await axios.get('/api/payment/status', {
      params: { order_no: response.data.data.order_no }
    });
    
    if (statusRes.data.data.status === 'success') {
      clearInterval(timer);
      // 支付成功，跳转
      window.location.href = '/payment-success';
    }
  }, 2000);
}
```

## 配置要求

### 1. 数据库配置
确保 `payment_config` 表中有微信支付配置：

```sql
SELECT * FROM payment_config WHERE payment_method = 'wechat';
```

需要配置：
- `app_id` - 微信公众号/开放平台AppID
- `mch_id` - 商户号
- `api_key` - API密钥
- `notify_url` - 支付回调地址
- `is_enabled` - 设置为1启用

### 2. 微信公众号配置（JSAPI支付）
- 配置支付授权目录
- 配置JS接口安全域名
- 获取用户openid的授权回调域名

### 3. 微信开放平台配置（H5支付）
- 配置H5支付域名

## 开发模式

如果微信支付配置未启用（`is_enabled = 0`），系统会自动使用模拟支付：
- 生成模拟二维码
- 可通过 `/api/payment/mock-success` 接口模拟支付成功
- 不会真实扣款

## 常见问题

### Q: 如何强制使用某种支付方式？
A: 在请求中添加 `trade_type` 参数：
```javascript
{
  trade_type: 'jsapi'  // 强制使用JSAPI支付
}
```

### Q: JSAPI支付提示需要openid怎么办？
A: 需要先通过微信授权获取用户openid，参考微信公众号开发文档。

### Q: H5支付跳转后如何返回？
A: 在创建支付时传入 `redirect_url` 参数，支付完成后会跳转到该地址。

### Q: Native支付如何知道用户已支付？
A: 前端需要轮询 `/api/payment/status` 接口查询支付状态。
