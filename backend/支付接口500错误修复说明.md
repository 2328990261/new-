# 支付接口500错误修复说明

## 问题描述
POST `/api/payment/create` 接口返回500错误（Internal Server Error）

## 根本原因

从生产环境日志分析：
```
[2026-01-25T15:58:48+08:00][error] [0]Call to undefined method app\service\WechatPayService::createH5Payment()
```

**核心问题：生产环境的WechatPayService类缺少支付方法**

### 具体问题

1. **字段名称错误**
   - 代码中使用了 `customer_service` 字段
   - 但数据库表 `fa_payments` 中实际字段名是 `dispatcher_id`

2. **生产环境代码版本过旧**
   - deployment目录中的WechatPayService缺少以下方法：
     - `createH5Payment()` - H5支付
     - `createJsapiPayment()` - JSAPI支付  
     - `createNativePayment()` - Native扫码支付
   - 只有基础的 `createMockPayment()` 模拟支付方法

3. **PaymentConfig表可能不存在**
   - 代码直接调用 `PaymentConfig::isEnabled()` 可能导致异常

## 修复方案

### 1. 修正字段名称
```php
// 修改前
$data['customer_service'] = $data['staff_id'];

// 修改后
$data['dispatcher_id'] = $data['staff_id'];
```

### 2. 添加异常处理
```php
// 检查支付配置时添加try-catch
try {
    $useRealPayment = PaymentConfig::isEnabled($data['payment_method']);
} catch (\Exception $e) {
    // 如果配置表不存在，默认使用模拟支付
    $useRealPayment = false;
}
```

### 3. 改进WechatPayService构造函数
```php
public function __construct()
{
    try {
        $this->config = PaymentConfig::getConfig('wechat');
    } catch (\Exception $e) {
        $this->config = null;
        trace('获取微信支付配置失败: ' . $e->getMessage(), 'warning');
    }
}
```

### 4. 增强错误日志
```php
catch (\Exception $e) {
    trace('创建支付订单失败: ' . $e->getMessage(), 'error');
    trace('错误堆栈: ' . $e->getTraceAsString(), 'error');
    return json([
        'code' => 500,
        'message' => '创建失败：' . $e->getMessage(),
        'error' => $e->getMessage(),
        'file' => $e->getFile(),
        'line' => $e->getLine()
    ]);
}
```

### 5. 添加方法存在性检查（兼容旧版本）
```php
// 检查方法是否存在，如果不存在则使用模拟支付
if (!method_exists($wechatService, 'createH5Payment')) {
    trace('createH5Payment方法不存在，使用模拟支付', 'warning');
    $payResult = $wechatService->createMockPayment([...]);
} else {
    $payResult = $wechatService->createH5Payment([...]);
}
```

## 支付流程说明

### 模拟支付模式（默认）
- 当 `payment_config` 表不存在或 `is_enabled=0` 时自动启用
- 生成模拟二维码和支付链接
- 可通过 `/api/payment/mock-success` 接口模拟支付成功

### 真实支付模式
- 需要在 `payment_config` 表中配置微信支付参数
- 设置 `is_enabled=1` 启用
- 支持三种支付方式：
  - Native扫码支付（PC端）
  - JSAPI支付（微信内H5）
  - MWEB支付（手机浏览器）

## 注意事项

1. **生产环境部署前**必须配置真实的微信支付参数
2. 模拟支付仅用于开发测试，不能用于生产环境
3. 确保数据库表结构与模型定义一致
4. 定期检查错误日志，及时发现问题
