# 协议接口说明

## 问题描述

线上部署后，支付页面报错：
```
GET https://t.jiajiao91.com/api/api/agreement/payment 404 (Not Found)
```

## 原因分析

支付页面（Payment.vue）在加载时会请求协议接口，但该接口之前没有实现。

## 解决方案

### 1. 创建协议API控制器

文件：`new_system/backend/app/controller/api/Agreement.php`

提供以下接口：
- `/api/agreement/payment` - 获取支付协议
- `/api/agreement/teacher` - 获取教师注册协议
- `/api/agreement/user` - 获取用户服务协议
- `/api/agreement/privacy` - 获取隐私政策

### 2. 添加路由配置

在 `new_system/backend/route/api.php` 中添加：

```php
// 协议相关（公开接口）
Route::get('agreement/payment', 'api.Agreement/payment');
Route::get('agreement/teacher', 'api.Agreement/teacher');
Route::get('agreement/user', 'api.Agreement/user');
Route::get('agreement/privacy', 'api.Agreement/privacy');
```

### 3. 接口功能

#### 获取支付协议
```
GET /api/agreement/payment
```

**响应示例：**
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "id": 1,
    "type": "payment",
    "title": "91家教接单协议",
    "content": "<div>...</div>",
    "version": "1.0",
    "effective_time": "2024-01-01 00:00:00",
    "update_time": "2024-01-01 00:00:00"
  }
}
```

#### 获取教师注册协议
```
GET /api/agreement/teacher
```

#### 获取用户服务协议
```
GET /api/agreement/user
```

#### 获取隐私政策
```
GET /api/agreement/privacy
```

## 工作原理

### 1. 数据库查询

控制器会先从数据库的 `agreements` 表中查询对应类型的当前版本协议：

```php
$agreement = AgreementModel::where('type', 'payment')
    ->where('is_current', 1)
    ->where('status', 1)
    ->find();
```

### 2. 默认内容

如果数据库中没有找到协议，会返回默认的协议内容：

```php
if (!$agreement) {
    return json([
        'code' => 200,
        'message' => '获取成功',
        'data' => [
            'id' => 0,
            'type' => 'payment',
            'title' => '91家教接单协议',
            'content' => $this->getDefaultPaymentAgreement(),
            'version' => '1.0',
            'effective_time' => date('Y-m-d H:i:s'),
            'update_time' => date('Y-m-d H:i:s')
        ]
    ]);
}
```

### 3. 前端调用

支付页面在加载时会自动调用：

```javascript
const loadAgreement = async () => {
  try {
    const response = await request.get('/api/agreement/payment')
    if (response.code === 200 && response.data) {
      agreementContent.value = response.data.content
    }
  } catch (error) {
    // 使用默认协议内容
    console.log('使用默认协议内容')
  }
}
```

## 协议管理

### 后台管理

管理员可以在后台管理系统中管理协议：

1. **创建协议**
   - 选择协议类型（payment/teacher/user/privacy）
   - 输入协议标题和内容
   - 设置版本号
   - 设置是否为当前版本

2. **更新协议**
   - 修改协议内容
   - 更新版本号
   - 设置生效时间

3. **版本管理**
   - 支持多版本协议
   - 只有一个版本可以设为"当前版本"
   - 历史版本可以查看但不会显示给用户

### 数据库表结构

表名：`agreements`

主要字段：
- `id` - 协议ID
- `type` - 协议类型（payment/teacher/user/privacy）
- `title` - 协议标题
- `content` - 协议内容（HTML格式）
- `plain_content` - 纯文本内容
- `version` - 版本号
- `status` - 状态（1启用/0禁用）
- `is_current` - 是否当前版本（1是/0否）
- `effective_time` - 生效时间
- `create_time` - 创建时间
- `update_time` - 更新时间

## 常见问题

### Q1: 接口返回404

**原因：**
- 文件未上传
- 路由配置未生效
- 缓存未清除

**解决方法：**
1. 确认文件已上传到正确位置
2. 检查路由配置是否正确
3. 清除ThinkPHP缓存：`php think clear`
4. 重启PHP-FPM服务

### Q2: 接口返回500错误

**原因：**
- PHP语法错误
- 数据库连接失败
- 权限问题

**解决方法：**
1. 查看PHP错误日志
2. 检查数据库配置
3. 确认文件权限正确

### Q3: 协议内容显示不正确

**原因：**
- HTML格式问题
- 编码问题
- 前端渲染问题

**解决方法：**
1. 检查协议内容的HTML格式
2. 确认使用UTF-8编码
3. 检查前端的v-html渲染

### Q4: 如何自定义协议内容？

**方法1：通过后台管理**
1. 登录管理后台
2. 进入协议管理
3. 创建或编辑协议
4. 设置为当前版本

**方法2：直接修改代码**
1. 编辑 `Agreement.php`
2. 修改 `getDefaultPaymentAgreement()` 等方法
3. 重新部署
