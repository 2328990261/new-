# 登录页面头像上传修复说明

## 问题描述

用户在小程序登录页面选择头像后，头像仍然保存为临时路径 `http://tmp/xxx.jpeg`，而不是上传到服务器获得真实的头像URL。

从调试信息可以看到：
```
微信头像授权回调: {type: "chooseavatar", ...}
detail: {avatarUrl: "http://tmp/fyA5zeXmJMCf78e36143b3079b3652148f3ccb08d613.jpeg"}
```

用户信息中仍然保存的是临时路径：
```
用户信息: {avatar: "http://tmp/fyA5zeXmJMCf78e36143b3079b3652148f3ccb08d613.jpeg", ...}
```

## 问题分析

**原始代码问题**：
1. 登录页面的 `onChooseAvatar` 方法只是将临时头像URL保存到本地存储
2. 没有调用后端的头像上传接口
3. 导致用户登录后头像仍然是微信的临时路径

**影响**：
- 用户看到的头像是临时路径，可能无法正常显示
- 数据库中保存的也是临时路径，不是真实的头像文件
- 影响用户体验

## 修复方案

### 1. 修改登录页面头像上传逻辑

**文件**: `tjiajiao91-main/预约家教小程序/pages/login/index.vue`

**修复内容**:
1. **导入环境配置**: 添加 `import envConfig from '@/config/env.js'`
2. **新增头像上传方法**: `uploadAvatarToServer(avatarUrl)`
3. **修改头像选择回调**: 在 `onChooseAvatar` 中调用头像上传

### 2. 头像上传流程

```javascript
onChooseAvatar(e) {
    const { avatarUrl } = e.detail
    if (avatarUrl) {
        // 1. 先显示临时头像
        this.userAvatar = avatarUrl
        
        // 2. 显示上传提示
        uni.showLoading({ title: '上传头像中...' })
        
        // 3. 上传头像到服务器
        this.uploadAvatarToServer(avatarUrl)
    }
}
```

### 3. 服务器上传逻辑

```javascript
uploadAvatarToServer(avatarUrl) {
    // 1. 获取用户标识（openid或临时标识）
    const userInfo = uni.getStorageSync('userInfo') || {}
    let uploadOpenid = userInfo.openid || 'temp_' + Date.now()
    
    // 2. 上传文件到服务器
    uni.uploadFile({
        url: envConfig.API_BASE_URL + '/api/avatar/upload',
        filePath: avatarUrl,
        name: 'avatar',
        formData: { openid: uploadOpenid },
        success: (uploadRes) => {
            // 3. 处理上传结果
            const result = JSON.parse(uploadRes.data)
            if (result.code === 200) {
                // 4. 构建完整头像URL
                let fullAvatarUrl = result.data.avatar_url
                if (!fullAvatarUrl.startsWith('http')) {
                    fullAvatarUrl = envConfig.API_BASE_URL + '/' + fullAvatarUrl
                }
                
                // 5. 更新本地头像
                this.userAvatar = fullAvatarUrl
                
                // 6. 保存到本地存储
                const localUserInfo = uni.getStorageSync('userInfo') || {}
                localUserInfo.avatar = fullAvatarUrl
                uni.setStorageSync('userInfo', localUserInfo)
                
                uni.showToast({ title: '头像上传成功', icon: 'success' })
            }
        }
    })
}
```

## 技术细节

### 1. 用户标识处理
- 如果用户已登录，使用真实的 `openid`
- 如果用户未登录，生成临时标识 `temp_` + 时间戳
- 后端接口支持通过 `openid` 或 `user_id` 上传头像

### 2. URL构建逻辑
- 检查服务器返回的URL是否为完整URL
- 如果是相对路径，添加域名前缀
- 支持开发环境和生产环境的自动切换

### 3. 错误处理
- 上传失败时显示错误提示
- 保留详细的调试日志
- 网络错误时的重试提示

## 预期效果

修复后，用户在登录页面选择头像时：
1. 立即显示选择的头像（临时路径）
2. 自动上传到服务器
3. 上传成功后替换为真实的头像URL
4. 保存真实URL到本地存储
5. 后续登录时使用真实的头像URL

用户信息将显示：
```
用户信息: {avatar: "http://localhost:8000/uploads/avatars/202403/avatar_xxx.jpg", ...}
```

而不是临时路径：
```
用户信息: {avatar: "http://tmp/xxx.jpeg", ...}
```
