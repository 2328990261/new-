# 头像上传功能说明

## 功能概述
小程序用户管理界面的头像上传功能，支持管理员为用户上传和更换头像。

## 实现细节

### 前端实现 (MiniUserManage.vue)

#### 1. 头像显示
- 在用户列表中显示用户头像缩略图
- 在编辑对话框中显示当前头像预览
- 支持头像加载失败时显示默认头像（用户昵称首字母）

#### 2. 头像上传组件
- 使用 Element Plus 的 `el-upload` 组件
- 支持点击上传，显示上传进度
- 文件类型限制：JPG、PNG、GIF、WebP
- 文件大小限制：最大 2MB

#### 3. 上传参数
```javascript
uploadData: {
  openid: '',    // 用户OpenID（优先）
  user_id: ''    // 用户ID（备用）
}
```

#### 4. 上传流程
1. 用户点击"编辑"按钮，获取完整用户信息
2. 在编辑对话框中点击"点击上传头像"按钮
3. 选择图片文件，进行前端验证
4. 设置上传参数（优先使用openid，否则使用user_id）
5. 调用后端上传接口 `/admin/api/avatar/upload`
6. 上传成功后更新表单中的头像URL
7. 保存时将新头像URL和用户角色一起提交

### 后端实现 (AvatarUpload.php)

#### 1. 上传接口
- 路径：`POST /admin/api/avatar/upload`
- 支持通过 `openid` 或 `user_id` 识别用户
- 如果只提供 `user_id`，会自动查询对应的 `openid`

#### 2. 文件处理
- 验证文件类型和大小
- 生成唯一文件名：`avatar_md5(openid+时间戳).扩展名`
- 保存到：`uploads/avatars/年月/` 目录

#### 3. 数据库更新
- 同时更新 `fa_users` 表的 `avatar` 字段
- 同时更新 `fa_wechat_users` 表的 `headimgurl` 字段
- 确保数据一致性

#### 4. URL处理
- 开发环境：返回完整URL（包含域名）
- 生产环境：返回相对路径，由前端处理完整URL

### 用户信息更新 (MiniProgramUser.php)

#### 1. 更新接口
- 路径：`PUT /admin/api/mini-users/{id}`
- 支持更新：昵称、手机号、头像、用户角色

#### 2. 数据库操作
- 使用事务确保数据一致性
- `fa_users` 表：更新基本信息（昵称、手机号、头像）
- `fa_wechat_users` 表：更新用户角色（user_type）

## 路由配置

### 管理端路由 (admin.php)
```php
// 头像上传
Route::post('avatar/upload', 'api.AvatarUpload/upload');
Route::post('avatar/cleanup-temp', 'api.AvatarUpload/cleanupTemp');
```

### 前端API配置
- 基础URL: `/admin/api`
- 上传URL: `/admin/api/avatar/upload`

## 使用说明

### 管理员操作流程
1. 进入"小程序用户管理"页面
2. 找到需要修改头像的用户，点击"编辑"按钮
3. 在编辑对话框中，点击"点击上传头像"按钮
4. 选择图片文件（支持JPG、PNG、GIF、WebP，最大2MB）
5. 等待上传完成，预览新头像
6. 可同时修改用户角色（家长/老师/管理员）
7. 点击"保存"按钮完成修改

### 功能特点
1. **智能用户识别**：优先使用OpenID，备用用户ID
2. **实时预览**：上传成功后立即显示新头像
3. **数据同步**：同时更新用户表和微信用户表
4. **角色管理**：支持修改用户角色并显示不同颜色标签
5. **错误处理**：完善的错误提示和重试机制

### 注意事项
1. 头像文件会保存在服务器的 `uploads/avatars/` 目录下
2. 上传成功后会自动更新数据库中的头像URL
3. 支持的图片格式：JPG、PNG、GIF、WebP
4. 文件大小限制：最大2MB
5. 头像URL会同时更新到用户表和微信用户表

## 错误处理
- 文件类型不支持：提示用户选择正确格式
- 文件过大：提示用户压缩图片
- 上传失败：显示具体错误信息
- 网络错误：提示用户重试

## 调试信息
后端会记录详细的上传日志，包括：
- 接收到的参数
- 文件信息
- 保存路径
- 数据库更新结果
- 最终返回的URL

日志可以在后端日志文件中查看，便于排查问题。
