## 工作流程

```
用户登录成功
    ↓
显示位置授权弹窗
    ↓
用户点击"允许"
    ↓
打开微信位置选择
    ↓
获取位置信息（经纬度、地址、省市区）
    ↓
调用API保存到数据库
    ↓
更新本地存储
    ↓
完成登录流程
```

## 数据库字段

在 `fa_wechat_users` 表中添加了以下字段：
- `latitude` - 纬度 (decimal 10,7)
- `longitude` - 经度 (decimal 10,7)
- `address` - 详细地址 (varchar 500)
- `province` - 省份 (varchar 50)
- `city` - 城市 (varchar 50)
- `district` - 区县 (varchar 50)

以及两个索引：
- `idx_location` - 经纬度索引
- `idx_city` - 城市区县索引

## API接口

### POST /api/user_location.php
保存用户位置信息

请求参数：
```json
{
  "openid": "用户openid",
  "latitude": 23.1291,
  "longitude": 113.2644,
  "address": "广东省广州市天河区天河路123号",
  "province": "广东省",
  "city": "广州市",
  "district": "天河区"
}
```

响应：
```json
{
  "code": 200,
  "success": true,
  "message": "位置信息保存成功",
  "data": { ... }
}
```
