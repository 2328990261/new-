# 邮件通知问题排查指南

## 问题现象
管理员分享小程序给用户，用户提交订单后，管理员没有收到邮件通知。

## 排查步骤

### 1. 检查管理员邮箱是否设置
```sql
-- 运行 check_and_update_admin_email.sql
-- 查看你的管理员账号是否有邮箱地址
SELECT id, username, nickname, email, openid FROM fa_admin WHERE openid = 'opYqv7QqN1EN-hMFhsG5O9o3f1Gw';
```

如果 email 字段为空，需要设置邮箱：
```sql
UPDATE fa_admin 
SET email = '你的邮箱@example.com'
WHERE openid = 'opYqv7QqN1EN-hMFhsG5O9o3f1Gw';
```

### 2. 检查邮件配置是否启用
```sql
-- 运行 check_email_config.sql
SELECT email_enabled, smtp_host, smtp_username FROM fa_notification_config WHERE id = 1;
```

如果 email_enabled = 0，需要启用：
```sql
UPDATE fa_notification_config SET email_enabled = 1 WHERE id = 1;
```

### 3. 检查邮件日志
访问管理后台的"邮箱日志"页面（http://localhost:3002/#/email-logs），查看：
- 是否有发送记录
- 发送状态（成功/失败）
- 如果失败，查看错误信息

### 4. 查看后端日志
后端日志会记录详细的邮件发送过程：
- 是否找到管理员
- 管理员是否有邮箱
- 邮件发送是否成功

日志位置：`tjiajiao91-main/new_system/backend/runtime/log/`

### 5. 测试邮件功能
在管理后台的"通知配置"页面，点击"发送测试邮件"按钮，测试邮件配置是否正确。

## 常见问题

### 问题1：管理员没有设置邮箱
**解决方案**：在管理后台的"管理员管理"页面，编辑管理员信息，添加邮箱地址。

### 问题2：邮件配置未启用
**解决方案**：在管理后台的"通知配置"页面，启用邮件通知功能。

### 问题3：SMTP配置错误
**解决方案**：
- 检查 SMTP 服务器地址、端口、用户名、密码是否正确
- QQ邮箱需要使用授权码，不是登录密码
- 163邮箱也需要使用授权码

### 问题4：邮件发送失败
**解决方案**：
- 查看邮件日志中的错误信息
- 检查服务器是否能访问 SMTP 服务器
- 检查防火墙是否阻止了 SMTP 端口（465或587）
