# 快速修复404错误指南

## 错误现象

```
POST https://t.jiajiao91.com/api/api/payment/create 404 (Not Found)
GET https://t.jiajiao91.com/api/api/agreement/payment 404 (Not Found)
```

注意：URL中出现了两个 `/api/`

## 快速修复

### 步骤1：修复前端代码

已修复的文件：
- ✅ `Payment.vue` - 支付页面
- ✅ `TeacherRegister.vue` - 教师注册页面

### 步骤2：重新编译

```bash
cd new_system/frontend/user
npm run build
```

### 步骤3：上传文件

将 `dist/` 目录下的所有文件上传到服务器的 `public/` 目录。

### 步骤4：清除缓存

**方法1：强制刷新浏览器**
- Windows: `Ctrl + F5`
- Mac: `Cmd + Shift + R`

**方法2：清除浏览器缓存**
1. 打开浏览器设置
2. 清除浏览历史记录
3. 选择"缓存的图片和文件"
4. 清除数据

### 步骤5：测试验证

访问支付页面：
```
https://yourdomain.com/payment
```

打开浏览器控制台（F12），检查：
- ✅ 没有404错误
- ✅ API请求路径正确（只有一个 `/api/`）
- ✅ 页面功能正常

## 问题原因

`request.js` 中已经配置了 `baseURL: '/api'`，但在发送请求时又加了 `/api/` 前缀，导致路径重复。

**错误：**
```javascript
request.post('/api/payment/create')  // → /api/api/payment/create ❌
```

**正确：**
```javascript
request.post('/payment/create')      // → /api/payment/create ✅
```

## 完成！

修复完成后，所有API请求应该正常工作。
