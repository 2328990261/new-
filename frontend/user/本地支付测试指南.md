# 本地支付功能测试指南

## 测试流程概述

前端支付页面会调用后端API创建支付订单，然后跳转到微信支付页面。

## 测试步骤

### 1. 准备工作

确保以下服务正常运行：
- 后端服务：`http://localhost:8088` 或你的后端地址
- 前端用户端：`http://localhost:5173` 或你的前端地址
- 数据库服务正常
- 微信支付配置已完成（在管理后台配置）

### 2. 访问支付页面

打开浏览器访问：
```
http://localhost:5173/payment
```

### 3. 填写支付信息

在支付页面填写：
- **信息费金额**：输入测试金额，如 `0.01`（1分钱）
- **真实姓名**：输入测试姓名
- **家教信息**：输入测试信息，如"南山地铁站初三数学"
- **派单客服**：选择任意客服
- **同意协议**：勾选协议复选框

### 4. 点击"立即支付"

点击按钮后，系统会：
1. 验证表单数据
2. 调用后端API：`POST /api/payment/create`
3. 后端创建支付订单并调用微信支付API
4. 返回微信H5支付URL
5. 前端跳转到微信支付页面

### 5. 查看网络请求

打开浏览器开发者工具（F12），切换到 Network 标签：

**请求信息：**
- URL: `http://localhost:8088/api/payment/create`
- Method: POST
- Request Body:
```json
{
  "amount": 0.01,
  "real_name": "测试用户",
  "tutor_info": "南山地铁站初三数学",
  "staff_id": 1,
  "agree_terms": true,
  "payment_method": "wechat",
  "trade_type": "h5",
  "redirect_url": "http://localhost:5173/payment-success"
}
```

**期望响应：**
```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "order_no": "订单号",
    "mweb_url": "https://wx.tenpay.com/cgi-bin/mmpayweb-bin/checkmweb?...",
    "payment_url": "微信支付URL"
  }
}
```

### 6. 测试场景

#### 场景1：正常支付流程
1. 填写完整信息
2. 点击支付
3. 跳转到微信支付页面
4. 使用微信扫码支付
5. 支付成功后跳转到成功页面

#### 场景2：表单验证
测试各个必填项的验证：
- 不填金额 → 提示"请输入支付金额"
- 不填姓名 → 提示"请输入真实姓名"
- 不填家教信息 → 提示"请输入家教信息"
- 不选客服 → 提示"请选择派单客服"
- 不勾选协议 → 提示"请先阅读并同意协议"

#### 场景3：API错误处理
- 后端服务未启动 → 提示"支付请求失败"
- 配置错误 → 显示后端返回的错误信息
- 网络超时 → 提示"支付请求失败"

### 7. 检查后端日志

查看后端控制台输出，确认：
- 接收到支付请求
- 调用微信支付API
- 返回支付URL

### 8. 检查数据库

查询 `payments` 表，确认：
- 订单记录已创建
- 订单状态为 `pending`（待支付）
- 金额、用户信息等字段正确

```sql
SELECT * FROM payments ORDER BY create_time DESC LIMIT 1;
```

### 9. 模拟支付成功

如果无法真实支付，可以手动修改数据库：

```sql
UPDATE payments 
SET status = 'paid', 
    paid_time = NOW() 
WHERE order_no = '你的订单号';
```

然后访问成功页面：
```
http://localhost:5173/payment-success
```

## 常见问题

### 问题1：点击支付后没有跳转
**检查：**
- 浏览器控制台是否有错误
- Network标签中API请求是否成功
- 响应数据中是否有 `mweb_url`

### 问题2：跳转到微信支付后显示错误
**可能原因：**
- 微信支付配置不正确（AppID、商户号、API密钥）
- 回调URL配置错误
- 金额格式不正确

### 问题3：支付成功但未跳转回来
**检查：**
- `redirect_url` 是否正确
- 微信支付回调是否正常
- 后端回调接口是否正常处理
