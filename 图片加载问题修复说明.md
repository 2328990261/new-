# 图片加载问题修复说明

## 问题描述
管理后台的教师详情页面中，教学风采照片和认证材料图片无法加载显示。

## 问题原因
数据库中存储的图片路径是相对路径（如 `/uploads/teacher/xxx.jpg`），前端直接使用这些相对路径会导致图片无法加载。需要拼接完整的后端服务器地址。

## 修复内容

### 1. 添加图片 URL 处理函数
在 `TeacherDetail.vue` 中添加了 `getImageUrl` 方法：

```javascript
// 获取完整的图片 URL
const getImageUrl = (path) => {
  if (!path) return ''
  // 如果已经是完整 URL，直接返回
  if (path.startsWith('http://') || path.startsWith('https://')) {
    return path
  }
  // 拼接后端服务器地址
  const backendUrl = import.meta.env.VITE_BACKEND_URL || 'http://localhost:8080'
  // 确保路径以 / 开头
  const imagePath = path.startsWith('/') ? path : `/${path}`
  return `${backendUrl}${imagePath}`
}
```

### 2. 修复教学风采照片显示
```vue
<el-image
  v-for="(photo, index) in teacher.teaching_photos"
  :key="index"
  :src="getImageUrl(photo)"
  :preview-src-list="teacher.teaching_photos.map(p => getImageUrl(p))"
  fit="cover"
  style="width: 120px; height: 120px; border-radius: 8px; cursor: pointer; border: 2px solid #e4e7ed;"
/>
```

### 3. 修复认证材料图片显示
- 身份证正面/反面：`:src="getImageUrl(teacher.id_card_front)"`
- 学历证明：`:src="getImageUrl(teacher.education_certificate)"`
- 教师资格证：`:src="getImageUrl(teacher.teacher_certificate)"`

### 4. 修复头像显示
```vue
<el-avatar :src="getImageUrl(teacher.avatar)" :size="80">
  {{ teacher.name?.charAt(0) || '?' }}
</el-avatar>
```

### 5. 更新环境配置
修改了 `new_system/frontend/admin/.env.development`：
```env
VITE_BACKEND_URL=http://localhost:8080
```

将后端地址从 8088 改为 8080，与实际启动的后端服务端口一致。

## Vite 代理配置
`vite.config.js` 中已配置 `/uploads` 路径的代理：
```javascript
proxy: {
  '/uploads': {
    target: 'http://localhost:8080',
    changeOrigin: true
  }
}
```

### 本地开发环境
- 后端服务：http://localhost:8080
- 图片路径：`/uploads/teacher/xxx.jpg`
- 完整 URL：`http://localhost:8080/uploads/teacher/xxx.jpg`
- 通过 Vite 代理访问

## 注意事项

1. **环境变量更新**：修改 `.env.development` 后需要重启前端服务
2. **图片路径格式**：数据库中存储的路径应该是 `/uploads/...` 格式
3. **跨域问题**：通过 Vite 代理解决，生产环境需要配置 Nginx 代理
4. **图片不存在**：如果图片文件不存在，会显示 Element Plus 的默认占位图
