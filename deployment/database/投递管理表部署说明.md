# 投递管理表部署说明

## 问题描述

线上环境出现两个错误：

### 错误1：表不存在
```
SQLSTATE[42S02]: Base table or view not found: 1146 Table 'tjiajiao91.fa_resume_application' doesn't exist
```

### 错误2：字段不存在
```
SQLSTATE[42S22]: Column not found: 1054 Unknown column 'ra.tutor_id' in 'on clause'
```

## 原因分析

1. 之前创建的表名是 `fa_teacher_applications`，但系统代码使用的是 `fa_resume_application`
2. 表中的字段名是 `tutor_order_id`，但控制器代码使用的是 `tutor_id`

## 解决方案

### 方案A：先检查表结构（推荐）

1. 先执行检查SQL，查看当前表结构：
```bash
mysql -u用户名 -p密码 数据库名 < deployment/database/check_table_structure.sql
```

2. 根据检查结果，选择对应的修复方案：

   - 如果表中有 `tutor_order_id` 字段 → 执行 `step1_rename_tutor_order_id.sql`
   - 如果表中已经是 `tutor_id` 字段 → 跳过步骤1
   - 如果缺少 `reviewer_id` 字段 → 执行 `直接添加缺失字段.sql`

### 方案B：智能修复（自动检测）

使用智能SQL，自动检测并修复所有问题：

```bash
mysql -u用户名 -p密码 数据库名 < deployment/database/smart_fix.sql
```

这个SQL会自动：
1. 检查 `tutor_order_id` 是否存在，如果存在则重命名为 `tutor_id`
2. 检查并添加缺失的 `reviewer_id` 字段
3. 检查并添加缺失的 `apply_time` 字段
4. 检查并添加缺失的 `review_time` 字段
5. 添加相关索引

### 方案C：全新创建（删除旧表）

如果以上方案都不行，删除旧表重新创建：

```sql
-- 删除旧表（会丢失数据！）
DROP TABLE IF EXISTS `fa_resume_application`;
DROP TABLE IF EXISTS `fa_teacher_applications`;
```

然后执行：
```bash
mysql -u用户名 -p密码 数据库名 < deployment/database/create_resume_application_table.sql
```

## 表结构说明

### 表名：fa_resume_application

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int(11) | 投递记录ID（主键） |
| teacher_id | int(11) | 教师ID |
| tutor_id | int(11) | 家教订单ID（注意：不是tutor_order_id） |
| status | enum | 状态：pending/approved/rejected |
| apply_time | datetime | 投递时间 |
| review_time | datetime | 审核时间 |
| admin_remark | text | 管理员备注 |
| reviewer_id | int(11) | 审核人ID |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 索引
- PRIMARY KEY: id
- INDEX: teacher_id, tutor_id, status, apply_time, reviewer_id
- UNIQUE INDEX: (teacher_id, tutor_id) - 防止重复投递

## 注意事项

1. 执行前请备份数据库
2. 必须按顺序执行：先重命名表，再修复字段名
3. 如果有数据，使用方案一（重命名+修复字段）
4. 如果是全新部署，使用方案二（创建新表）
5. 字段名必须是 `tutor_id`，不能是 `tutor_order_id`
6. 执行后清除ThinkPHP缓存：删除 `deployment/backend/runtime/` 目录

## 常见问题

### Q: 执行SQL后仍然报错？
A: 请检查：
1. 是否在正确的数据库中执行（tjiajiao91）
2. 字段名是否正确（tutor_id 不是 tutor_order_id）
3. 清除ThinkPHP缓存：删除 `deployment/backend/runtime/` 目录
4. 重启PHP-FPM或Web服务器

### Q: 不确定当前表结构？
A: 执行以下SQL查看：
```sql
DESCRIBE fa_resume_application;
```
如果看到 `tutor_order_id` 字段，说明需要执行步骤2修复字段名。

### Q: 需要删除表重新创建？
A: 执行以下SQL：
```sql
DROP TABLE IF EXISTS `fa_resume_application`;
DROP TABLE IF EXISTS `fa_teacher_applications`;
```
然后执行 `deployment/database/create_resume_application_table.sql`
