# 用户表修复步骤 - 简化版

## 问题说明
用户尝试执行 SQL 时遇到 phpMyAdmin 错误，原因是 nickname 字段已经存在。

## 解决方案

### 步骤1：检查表结构

在 phpMyAdmin 中打开 `myjiajiao` 数据库，点击 SQL 标签，执行：

```sql
DESC `users`;
```

这会显示 users 表的所有字段。记下哪些字段已存在：
- 如果有 `nickname` - 说明昵称字段已存在 ✓
- 如果有 `avatar` - 说明头像字段已存在 ✓
- 如果有 `platform` - 说明端口字段已存在 ✓

### 步骤2：只添加缺失的字段

根据步骤1的结果，只执行需要的 SQL：

#### 如果缺少 avatar 字段，执行：
```sql
ALTER TABLE `users` 
ADD COLUMN `avatar` VARCHAR(500) DEFAULT NULL COMMENT '头像URL' 
AFTER `phone`;
```

#### 如果缺少 platform 字段，执行：
```sql
ALTER TABLE `users` 
ADD COLUMN `platform` VARCHAR(20) DEFAULT 'miniprogram' COMMENT '用户端口' 
AFTER `avatar`;
```

#### 更新现有数据（这条总是要执行）：
```sql
UPDATE `users` 
SET `platform` = 'miniprogram' 
WHERE `platform` IS NULL OR `platform` = '';
```

### 步骤3：验证表结构

再次执行：
```sql
DESC `users`;
```

确认表中包含以下字段：
- id
- openid
- phone
- nickname
- avatar
- platform
- create_time
- update_time

### 步骤4：清空测试数据（可选）

如果想重新测试，可以清空现有用户：
```sql
TRUNCATE TABLE `users`;
```

### 步骤5：测试小程序登录

1. 打开小程序
2. 进入登录页面
3. 点击"用微信头像"按钮，选择头像
4. 输入昵称
5. 同意协议
6. 授权手机号登录

### 步骤6：检查数据

登录成功后，在 phpMyAdmin 中查看：
```sql
SELECT id, openid, phone, nickname, avatar, platform, create_time 
FROM `users` 
ORDER BY create_time DESC 
LIMIT 10;
```

应该能看到新用户的完整信息。

## 常见错误处理

### 错误 #1060: Duplicate column name 'xxx'
这说明字段已经存在，跳过这条 SQL 即可。

### 错误 #1054: Unknown column 'xxx'
这说明字段不存在，需要先添加该字段。

### phpMyAdmin 报错
如果 phpMyAdmin 出现其他错误，可以尝试：
1. 刷新页面重试
2. 一次只执行一条 SQL
3. 使用命令行工具连接数据库执行

## 调试方法

### 查看后端日志
```
new_system/backend/runtime/log/
```
查找包含"登录参数"的日志，确认后端是否接收到 nickname 和 avatar。

### 查看前端 Console
在小程序开发工具中查看登录请求的参数。
