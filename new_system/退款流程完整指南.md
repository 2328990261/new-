# 退款流程完整指南

## 概述

本系统已实现完整的退款申请和处理流程，包括用户端退款申请和管理端退款处理，支持微信支付自动退款。

---

## 一、功能特性

### 1. 用户端功能

- ✅ 在线退款申请
- ✅ 订单信息验证
- ✅ 退款进度查询
- ✅ 退款原因说明
- ✅ 退款金额自定义

### 2. 管理端功能

- ✅ 退款申请列表
- ✅ 退款状态筛选
- ✅ 退款数据统计
- ✅ 一键退款处理
- ✅ 退款申请驳回
- ✅ 微信自动退款

### 3. 退款状态

| 状态 | 说明 | 用户操作 | 管理员操作 |
|------|------|----------|------------|
| pending | 退款待处理 | 等待处理 | 可处理/驳回 |
| processing | 退款处理中 | 等待完成 | - |
| rejected | 退款已驳回 | 可重新申请 | - |
| completed | 退款已完成 | - | - |

## 二、使用流程

### 用户端：申请退款

#### 步骤1：访问退款申请页面
```
URL: http://你的域名/refund
```

#### 步骤2：输入订单信息
- 支付订单号（如：PAY20251004123456001）
- 支付人联系方式（手机号或邮箱）

#### 步骤3：填写退款信息
- 退款金额（不超过可退金额）
- 退款原因（至少10个字符）

#### 步骤4：提交申请
- 系统自动验证
- 发送邮件通知管理员
- 显示申请成功提示

#### 步骤5：查询退款状态
- 在申请成功页面点击"查询退款状态"
- 或重新访问退款申请页面查询

---

### 管理端：处理退款

#### 步骤1：登录管理后台
```
URL: http://你的域名/admin
```

#### 步骤2：进入支付管理模块
- 点击左侧菜单"支付管理"
- 点击"退款待处理"标签页

#### 步骤3：查看退款申请
- 查看申请人信息
- 查看退款原因
- 查看订单详情

#### 步骤4：处理退款
**方式一：同意退款**
1. 点击"退款"按钮
2. 确认退款金额（可修改）
3. 填写备注（可选）
4. 点击"确定退款"
5. 系统自动调用微信退款API（如果是微信支付）
6. 更新退款状态为"已退费"

**方式二：驳回退款**
1. 点击"查拒"按钮
2. 填写驳回原因（必填）
3. 点击"确定驳回"
4. 更新退款状态为"退款驳回"

## 三、微信支付退款接口

### 1. 配置要求

#### 必需配置项
- AppID：微信应用ID
- 商户号：微信商户号
- API密钥：微信API密钥（32位）
- 证书路径：微信支付证书文件路径（退款必需）
- 回调地址：支付回调URL

#### 证书配置
证书文件应放置在：
```
new_system/backend/cert/wechat/apiclient_cert.pem
```

在支付配置中设置证书路径：
```
cert/wechat/apiclient_cert.pem
```

### 2. 退款流程

```
用户申请退款
    ↓
管理员处理退款
    ↓
系统生成退款单号 (REF20251004123456001)
    ↓
调用微信退款API
    ↓
微信处理退款 (1-3个工作日)
    ↓
退款到账 (原路返回)
```

### 3. 退款API参数

```php
$refundData = [
    'order_no' => '商户订单号',
    'transaction_id' => '微信交易号',
    'refund_no' => '商户退款单号',
    'total_amount' => 订单总金额（元）,
    'refund_amount' => 退款金额（元）,
    'refund_reason' => '退款原因'
];
```

### 4. 返回结果

**成功：**
```json
{
    "success": true,
    "message": "退款成功",
    "data": {
        "refund_id": "微信退款单号",
        "out_refund_no": "商户退款单号",
        "refund_fee": 退款金额,
        "refund_channel": "退款渠道"
    }
}
```

**失败：**
```json
{
    "success": false,
    "message": "错误信息"
}
```

## 五、API接口文档

### 1. 获取支付记录（用于申请退款）

**接口：** `GET /api/refund/payment`

**参数：**
- order_no：订单号
- payer_contact：支付人联系方式

**返回：**
```json
{
    "success": true,
    "data": {
        "id": 1,
        "order_no": "PAY20251004123456001",
        "tutor_name": "家教名称",
        "amount": 100.00,
        "refunded_amount": 0.00,
        "can_refund_amount": 100.00,
        "paid_time": "2025-10-04 12:34:56",
        "refund_status": null,
        "refund_status_text": "无退款"
    }
}
```

---

### 2. 申请退款

**接口：** `POST /api/refund/apply`

**参数：**
- order_no：订单号
- payer_contact：支付人联系方式
- refund_amount：退款金额
- refund_reason：退款原因

**返回：**
```json
{
    "success": true,
    "message": "退款申请已提交",
    "data": {
        "order_no": "PAY20251004123456001",
        "refund_amount": 100.00,
        "apply_time": "2025-10-04 12:34:56"
    }
}
```

---

### 3. 查询退款状态

**接口：** `GET /api/refund/status`

**参数：**
- order_no：订单号
- payer_contact：支付人联系方式

**返回：**
```json
{
    "success": true,
    "data": {
        "order_no": "PAY20251004123456001",
        "amount": 100.00,
        "refunded_amount": 100.00,
        "refund_status": "completed",
        "refund_status_text": "退款已完成",
        "refund_apply_amount": 100.00,
        "refund_apply_time": "2025-10-04 12:34:56",
        "refund_time": "2025-10-04 15:00:00",
        "refund_reason": "用户申请退款",
        "reject_reason": null
    }
}
```

---

### 4. 管理端：处理退款

**接口：** `POST /admin/payments/refund/process`

**参数：**
- id：支付记录ID
- refund_amount：退款金额
- remark：备注（可选）
- auto_refund：是否自动退款（1=是，0=否，默认1）

**返回：**
```json
{
    "code": 200,
    "message": "退款成功",
    "data": {
        "wechat_refund": true,
        "refund_info": {
            "refund_id": "微信退款单号",
            "out_refund_no": "REF20251004123456001",
            "refund_fee": 100.00
        }
    }
}
```

---

### 5. 管理端：驳回退款

**接口：** `POST /admin/payments/refund/reject`

**参数：**
- id：支付记录ID
- reject_reason：驳回原因（必填）

**返回：**
```json
{
    "code": 200,
    "message": "驳回成功"
}
```

## 七、常见问题

### 1. 用户端常见问题

**Q: 忘记订单号怎么办？**
A: 可以查看支付成功页面、支付凭证或联系客服查询。

**Q: 退款多久到账？**
A: 管理员处理后，微信退款通常需要1-7个工作日到账。

**Q: 可以部分退款吗？**
A: 可以，在申请时可以自定义退款金额。

**Q: 退款被驳回怎么办？**
A: 查看驳回原因，如有疑问可联系客服协商。

---

### 2. 管理端常见问题

**Q: 退款需要手动转账吗？**
A: 不需要，系统会自动调用微信退款API。

**Q: 如何关闭自动退款？**
A: 在处理退款时将 auto_refund 参数设为 0。

**Q: 退款失败怎么办？**
A: 检查微信支付配置和证书是否正确，或联系技术支持。

**Q: 可以修改退款金额吗？**
A: 可以，在处理退款时可以修改退款金额。

---

### 3. 技术问题

**Q: 证书文件放哪里？**
A: `new_system/backend/cert/wechat/apiclient_cert.pem`

**Q: 如何测试退款功能？**
A: 使用测试环境和微信支付沙箱环境进行测试。

**Q: 退款记录在哪里查看？**
A: 管理后台 → 支付管理 → 已退费标签页

**Q: 如何配置管理员通知邮箱？**
A: 编辑 `new_system/backend/config/app.php` 文件。

## 九、注意事项

### 1. 安全注意事项

- ✅ 订单号和联系方式双重验证
- ✅ 退款金额不能超过可退金额
- ✅ 支付状态必须为"已支付"
- ✅ 管理员操作需要登录认证
- ✅ 退款原因必须填写

### 2. 业务注意事项

- ✅ 退款原路返回至支付账户
- ✅ 部分退款不影响查看家教信息
- ✅ 退款后实收金额自动计算
- ✅ 支持多次退款（直到全额退款）
- ✅ 退款记录永久保存

### 3. 技术注意事项

- ✅ 微信退款需要证书
- ✅ 证书权限需要正确设置
- ✅ 路由缓存需要清除
- ✅ 日志记录完整详细
- ✅ 异常处理完善
