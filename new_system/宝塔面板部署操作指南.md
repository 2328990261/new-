# 宝塔面板部署操作指南

## 部署概述

将本地开发的家教信息管理系统部署到宝塔面板服务器，包括后端API、管理端前端、用户端前端三个部分。

## 一、服务器环境准备

### 1.1 安装宝塔面板

如果服务器还未安装宝塔，执行：

```bash
# CentOS
yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh

# Ubuntu/Debian
wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh
```

### 1.2 安装运行环境

在宝塔面板中安装：

- PHP 7.4 或 8.0（推荐7.4）
- MySQL 5.7 或 8.0
- Nginx 1.20+
- Node.js 16+（通过PM2管理器或直接安装）

### 1.3 PHP扩展安装

在"软件商店" -> "PHP 7.4" -> "设置" -> "安装扩展"中安装：

- fileinfo
- redis（可选）
- opcache（推荐）

## 二、数据库配置

### 2.1 创建数据库

1. 进入宝塔面板 -> "数据库"
2. 点击"添加数据库"
3. 填写信息：
   - 数据库名：myjiajiao
   - 用户名：myjiajiao_user
   - 密码：生成强密码并保存

### 2.2 导入数据库

1. 上传数据库文件到服务器
2. 进入phpMyAdmin
3. 依次导入以下SQL文件（按顺序）：
   - `new_system/database/create_tables.sql` - 基础表结构
   - `new_system/database/provinces.csv` - 省份数据
   - `new_system/database/cities.csv` - 城市数据
   - `new_system/database/areas.csv` - 区域数据
   - `new_system/database/payments.sql` - 支付表
   - `new_system/database/teachers.sql` - 教师表
   - `new_system/database/teacher_auth.sql` - 教师认证表
   - `new_system/database/email_tables.sql` - 邮件表
   - `new_system/database/upgrade_wechat_share_config.sql` - 微信分享配置
   - `new_system/database/upgrade_seo_config.sql` - SEO配置
   - `new_system/database/preset_teachers_data.sql` - 预设教师数据

或使用SQL命令导入：

```bash
mysql -u myjiajiao_user -p myjiajiao < create_tables.sql
```

### 2.3 创建管理员账号

执行SQL：


默认密码：password

## 三、后端部署

### 3.1 上传后端代码

1. 在宝塔面板创建网站（例如：api.yourdomain.com）
2. 上传 `new_system/backend/` 目录内所有文件到网站根目录
3. 设置运行目录为 `/public`
4. 设置伪静态为 "ThinkPHP"

### 3.2 配置数据库连接

编辑 `config/database.php`：

```php
'connections' => [
    'mysql' => [
        'type'       => 'mysql',
        'hostname'   => '127.0.0.1',
        'database'   => 'myjiajiao',
        'username'   => 'myjiajiao_user',
        'password'   => '你的数据库密码',
        'hostport'   => '3306',
        'charset'    => 'utf8mb4',
        'prefix'     => 'fa_',
    ],
],
```

### 3.3 设置目录权限

在SSH或宝塔终端执行：

```bash
cd /www/wwwroot/api.yourdomain.com
chmod -R 755 public
chmod -R 777 runtime
```

### 3.4 配置跨域（如果需要）

编辑 `app/middleware/Cors.php` 中的允许域名：

```php
'Access-Control-Allow-Origin' => 'https://www.yourdomain.com,https://admin.yourdomain.com',
```

### 3.5 测试后端API

访问：https://api.yourdomain.com/admin/info

应该返回JSON响应（未登录错误是正常的）

## 四、前端部署

### 4.1 本地构建用户端

在本地项目目录执行：

```bash
cd new_system/frontend/user
npm install
npm run build
```

构建完成后，dist目录包含所有静态文件

### 4.2 上传用户端

1. 在宝塔面板创建网站（例如：www.yourdomain.com）
2. 上传 `new_system/frontend/user/dist/` 目录内所有文件到网站根目录
3. 设置运行目录为 `/`

### 4.3 配置用户端Nginx伪静态

在宝塔面板 -> 网站 -> 设置 -> 配置文件中添加：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### 4.4 修改用户端API地址

在构建前修改 `new_system/frontend/user/src/api/config.js`：

```javascript
const request = axios.create({
  baseURL: 'https://api.yourdomain.com',  // 改为实际后端域名
  timeout: 10000,
  withCredentials: true
})
```

### 4.5 本地构建管理端

在本地项目目录执行：

```bash
cd new_system/frontend/admin
npm install
npm run build
```

### 4.6 上传管理端

1. 在宝塔面板创建网站（例如：admin.yourdomain.com）
2. 上传 `new_system/frontend/admin/dist/` 目录内所有文件到网站根目录
3. 设置运行目录为 `/`
4. 配置Nginx伪静态（同用户端）

### 4.7 修改管理端API地址

在构建前修改 `new_system/frontend/admin/src/api/config.js`：

```javascript
const request = axios.create({
  baseURL: 'https://api.yourdomain.com',  // 改为实际后端域名
  timeout: 10000,
  withCredentials: true
})
```

## 五、SSL证书配置

### 5.1 申请SSL证书

在宝塔面板 -> 网站 -> 设置 -> SSL中：

1. 选择"Let's Encrypt"
2. 勾选域名
3. 点击"申请"
4. 开启"强制HTTPS"

为三个域名都配置SSL：

- api.yourdomain.com
- www.yourdomain.com
- admin.yourdomain.com

### 5.2 配置HTTPS重定向

在网站设置中开启"强制HTTPS"

## 六、性能优化

### 6.1 开启PHP OPcache

在宝塔面板 -> PHP设置 -> 配置修改中设置：

```ini
opcache.enable=1
opcache.memory_consumption=128
opcache.max_accelerated_files=10000
```

### 6.2 开启Gzip压缩

在网站设置 -> 配置文件中确保有：

```nginx
gzip on;
gzip_min_length 1k;
gzip_comp_level 6;
gzip_types text/plain text/css text/javascript application/json application/javascript application/x-javascript application/xml;
```

### 6.3 配置浏览器缓存

在网站设置 -> 配置文件中添加：

```nginx
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

### 7.1 后端环境配置

编辑 `.env` 文件（如果使用）或直接在 `config/app.php` 中配置：


- 回调域名

### 7.3 邮件配置

在管理端后台 -> 通知配置 -> 邮件配置中填写：

- SMTP服务器
- SMTP端口
- 发件人邮箱
- 密码/授权码

## 八、安全加固

### 8.1 关闭目录浏览

在网站设置 -> 配置文件中确保：

```nginx
autoindex off;
```

### 8.2 禁止访问敏感文件

添加以下规则：

```nginx
location ~* \.(env|log|sql|md|json|lock)$ {
    deny all;
}
```

### 8.3 设置防火墙规则

在宝塔面板 -> 安全 中：

- 开启SSH端口保护
- 配置常用端口（80、443、3306等）
- 设置禁止访问的IP（如有）

### 8.4 修改默认管理员密码

首次登录后立即修改 admin 账号密码

## 十、常见问题解决

### 10.1 500错误

**可能原因：**
- 目录权限不正确
- PHP扩展缺失
- 数据库连接失败

**解决方案：**
```bash
# 检查目录权限
chmod -R 755 /www/wwwroot/api.yourdomain.com
chmod -R 777 /www/wwwroot/api.yourdomain.com/runtime

# 检查PHP错误日志
tail -f /www/wwwroot/api.yourdomain.com/runtime/log/error.log
```

### 10.2 跨域问题

**解决方案：**
1. 检查 `app/middleware/Cors.php` 配置
2. 确保前端API地址正确
3. 检查Nginx配置

### 10.3 数据库连接失败

**解决方案：**
1. 检查数据库配置信息
2. 确认数据库用户权限
3. 检查防火墙设置

### 10.4 前端页面空白

**解决方案：**
1. 检查浏览器控制台错误
2. 确认API地址配置正确
3. 检查Nginx伪静态配置
