# 批量导出功能快速测试指南

## 前提条件

1. 确保已安装依赖：
```bash
cd new_system/frontend/admin
npm install
```

2. 启动前端开发服务器：
```bash
npm run dev
```

3. 确保后端服务正常运行

## 测试步骤

### 测试1：基本导出功能

1. 登录管理后台
2. 进入"家教信息管理"页面
3. 选择2-3条家教信息（不同城市）
4. 点击底部的"批量导出"按钮
5. 在对话框中选择"Word文档"
6. 点击"开始导出"
7. 检查下载目录，确认文件已生成

**预期结果**：
- 生成与选择的城市数量相同的Word文件
- 文件名格式正确：`城市名_家教单_日期时间.docx`
- 文件可以正常打开并查看内容

### 测试2：深圳区域分组

1. 选择多条深圳的家教信息（不同区域）
2. 点击"批量导出"
3. 查看统计信息，确认"深圳区域"数量正确
4. 选择"文本文件"格式
5. 开始导出

**预期结果**：
- 深圳的订单按区域分别导出
- 文件名格式：`深圳-区域名_家教单_日期时间.txt`
- 每个区域一个文件

### 测试3：混合城市导出

1. 选择包含：
   - 2条广州的订单
   - 3条深圳南山区的订单
   - 2条深圳福田区的订单
   - 1条北京的订单
2. 点击"批量导出"
3. 查看统计：
   - 非深圳城市：2（广州、北京）
   - 深圳区域：2（南山区、福田区）
   - 总文件数：4
4. 选择"两种格式都导出"
5. 开始导出

**预期结果**：
- 生成8个文件（4个Word + 4个TXT）
- 文件命名正确
- 内容完整

### 测试4：大量数据导出

1. 点击"全选当页"或"全选全部"
2. 点击"批量导出"
3. 观察导出进度
4. 等待所有文件下载完成

**预期结果**：
- 进度条正常显示
- 所有文件成功下载
- 浏览器可能提示"允许下载多个文件"

### 测试5：导出内容验证

打开导出的Word文档，检查：
- ✓ 标题显示正确
- ✓ 统计信息准确
- ✓ 每条订单包含：
  - 订单编号
  - 城市、区域、年级、科目、薪资
  - 详细内容
- ✓ 格式美观

打开导出的TXT文件，检查：
- ✓ 内容完整
- ✓ 中文显示正常（UTF-8编码）
- ✓ 分隔线清晰

## 常见问题排查

### 问题1：点击"批量导出"没有反应
**可能原因**：未选择任何家教信息
**解决方法**：先勾选至少一条家教信息

### 问题2：浏览器阻止下载
**现象**：提示"此网站正在尝试下载多个文件"
**解决方法**：点击"允许"或在浏览器设置中允许该网站的多文件下载

### 问题3：导出的Word文档打开失败
**可能原因**：
1. 文件下载不完整
2. Office版本过低

**解决方法**：
1. 重新导出
2. 使用WPS或新版Office打开
3. 尝试导出TXT格式

### 问题4：中文乱码
**可能原因**：文本编辑器编码问题
**解决方法**：使用支持UTF-8的编辑器（如VS Code、Notepad++）

### 问题5：导出进度卡住
**解决方法**：
1. 刷新页面重试
2. 检查浏览器控制台是否有错误
3. 减少选择的数据量
