# 小程序简历预览页面修复说明

## 问题描述
小程序端的简历预览页面无法显示教师信息，显示"未填写"和"暂无教学经历"。

## 问题原因
简历预览页面（`pages/teacher-resume-preview/index.vue`）在接收 `teacher_id` 参数时，没有调用后端 API 加载教师数据，导致页面无法显示教师信息。

## 修复内容

### 1. 添加数据加载功能
在 `onLoad` 方法中添加了从后端加载教师数据的逻辑：
- 如果有 `teacher_id` 参数，调用 `loadTeacherData()` 方法从后端加载数据
- 如果有 `data` 参数（从注册页面跳转），使用传递的数据并合并

### 2. 新增 `loadTeacherData` 方法
```javascript
async loadTeacherData(teacherId) {
    // 调用 API 获取教师详情
    const res = await teacherRegisterApi.getTeacherDetail(teacherId)
    // 填充简历数据
    this.resumeData = { ...teacher }
}
```

### 3. 添加状态管理
新增以下数据字段：
- `readonly`: 是否只读模式
- `reviewStatus`: 审核状态（pending/approved/rejected）
- `teacherId`: 教师ID

### 4. 优化底部按钮显示
根据不同场景显示不同的按钮：
- **新注册场景**（无 teacher_id）：显示"返回修改"和"提交注册"
- **编辑场景**（有 teacher_id）：显示"返回修改"和"编辑资料"
- **只读场景**（readonly=true）：只显示"返回"按钮
- **驳回场景**（rejected）：显示"返回修改"和"编辑资料"

### 5. 添加审核状态提示
在页面顶部显示审核状态提示卡片：
- **审核中**：黄色提示，显示"您的资料正在审核中，请耐心等待"
- **审核通过**：绿色提示，显示"恭喜您，您的资料已通过审核"
- **审核未通过**：红色提示，显示"您的资料未通过审核，请修改后重新提交"

### 6. 新增 `goToEdit` 方法
点击"编辑资料"按钮时，跳转到教师注册页面的编辑模式：
```javascript
goToEdit() {
    uni.redirectTo({
        url: `/pages/teacher-register/index?mode=edit&teacher_id=${this.teacherId}&fromPreview=true`
    })
}
```

## 后端 API
使用的后端接口：
- **路由**: `GET /api/teacher/detail/:id`
- **控制器**: `app\controller\api\Teacher::detail()`
- **返回数据**: 包含教师的所有信息（姓名、性别、学校、专业、教学经历、优势标签等）

## 使用场景

### 场景1：从个人中心查看简历
```javascript
// 跳转方式
uni.navigateTo({
    url: '/pages/teacher-resume-preview/index?teacher_id=123&readonly=true&status=pending'
})
```
- 显示审核状态提示
- 只显示"返回"按钮
- 从后端加载教师数据

### 场景2：从注册页面预览简历
```javascript
// 跳转方式
uni.navigateTo({
    url: `/pages/teacher-resume-preview/index?data=${encodeURIComponent(JSON.stringify(formData))}`
})
```
- 不显示审核状态提示
- 显示"返回修改"和"提交注册"按钮
- 使用传递的表单数据

### 场景3：审核被驳回后查看
```javascript
// 跳转方式
uni.navigateTo({
    url: '/pages/teacher-resume-preview/index?teacher_id=123&status=rejected&readonly=false'
})
```
- 显示红色的审核未通过提示
- 显示"返回修改"和"编辑资料"按钮
- 从后端加载教师数据
