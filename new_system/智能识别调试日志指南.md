# 智能识别调试日志指南

## 📋 如何查看调试日志

### 1. 打开浏览器开发者工具

- **Chrome/Edge**: 按 `F12` 或右键页面选择"检查"
- **快捷键**: `Ctrl + Shift + I` (Windows) 或 `Cmd + Option + I` (Mac)

### 2. 切换到 Console（控制台）选项卡

在开发者工具顶部找到 **Console** 标签并点击。

### 3. 清空之前的日志

点击控制台左上角的 🚫 清空按钮，清除之前的日志信息。

### 4. 执行识别操作

1. 点击"智能识别"按钮
2. 输入或粘贴家教信息文本
3. 点击"开始识别"

### 5. 查看日志输出

控制台会输出详细的调试日志，按照以下结构：

## 🔍 问题排查重点

### 1. 检查后端返回的数据

查看 `🎯 收到批量识别响应` 部分，确认：
- ✅ `result.city_id` 是否有值
- ✅ `result.district_id` 是否有值
- ✅ `result.subject_id` 是否有值
- ✅ `result.grade` 是否有值

### 2. 检查数据加载状态

查看 `📋 检查数据加载状态` 部分，确认：
- ✅ `formCities` 数量 > 0（城市列表已加载）
- ✅ `subjects` 数量 > 0（科目列表已加载）

### 3. 检查ID匹配过程

查看每个识别项的处理日志：

#### 城市匹配失败
```
🔍 查找城市 ID: 194
❌ 未找到城市 ID: 194
```
**可能原因**：
- 城市列表中没有该 ID
- ID 类型不匹配（数字 vs 字符串）

#### 区域匹配失败
```
🔍 查找区域 ID: 1829 城市ID: 194
区域列表返回: 0 个区域
❌ 未找到区域 ID: 1829
```
**可能原因**：
- 该城市下没有区域数据
- 区域 API 请求失败
- ID 不匹配

#### 科目匹配失败
```
🔍 查找科目 ID: 3
❌ 未找到科目 ID: 3
```
**可能原因**：
- 科目列表中没有该 ID
- ID 类型不匹配

### 4. 检查年级映射

查看年级映射日志：
```
📚 年级映射: 小六 → 小学六年级
```

如果显示：
```
📚 年级映射: 小六 → 小六
```
说明没有找到对应的映射，需要在 `gradeMapping` 中添加。

## 💡 常见问题与解决方案

### 问题1: 城市名称显示"未知"

**日志特征**：
```
❌ 未找到城市 ID: 194
```

**解决方案**：
1. 检查城市列表是否已加载
2. 检查 ID 类型是否匹配（可能需要用 `==` 而不是 `===`）
3. 在控制台输入 `formCities.value` 查看完整城市列表

### 问题2: 区域名称显示"未知"

**日志特征**：
```
区域列表返回: 0 个区域
```
或
```
❌ 加载区域失败: [错误信息]
```

**解决方案**：
1. 检查区域 API 是否正常
2. 检查该城市是否有区域数据
3. 检查网络请求是否成功

### 问题3: 科目名称显示"未知"

**日志特征**：
```
❌ 未找到科目 ID: 3
```

**解决方案**：
1. 检查科目列表是否已加载
2. 在控制台输入 `subjects.value` 查看完整科目列表
3. 检查 ID 是否匹配

### 问题4: 年级显示简写而不是完整名称

**日志特征**：
```
📚 年级映射: 小六 → 小六
```

**解决方案**：
在 `gradeMapping` 中添加对应的映射规则。

## 🔧 高级调试技巧

### 1. 导出日志

在控制台右键 → 选择"Save as..." 可以保存完整日志。

### 2. 过滤日志

在控制台的过滤框中输入关键词：
- `开始处理` - 只看主流程
- `❌` - 只看错误
- `✅` - 只看成功

### 3. 查看对象详情

点击日志中的对象（如 `{...}` ）可以展开查看完整内容。

### 4. 复制对象

右键对象 → "Copy object" 可以复制完整的 JSON 数据。

## 📞 问题反馈

如果遇到问题，请提供：
1. 完整的控制台日志截图
2. 输入的文本内容
3. 预期的结果 vs 实际的结果

这样可以更快定位问题！
