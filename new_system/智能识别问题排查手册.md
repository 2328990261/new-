# 智能识别问题排查手册

## 🔍 问题描述

您遇到的问题：
1. ❌ 控制台没有调试日志输出
2. ❌ 多个家教单文本被当作整个文本识别，没有分割

## ✅ 解决方案（按顺序执行）

### 方案 1: 强制刷新浏览器缓存 ⭐ 推荐首先尝试

#### Chrome/Edge 浏览器：
1. **按 Ctrl + Shift + Delete** 打开清除缓存窗口
2. 选择"缓存的图像和文件"
3. 时间范围选择"全部时间"
4. 点击"清除数据"
5. **按 Ctrl + F5** 强制刷新页面

#### 或者使用开发者工具强制刷新：
1. **按 F12** 打开开发者工具
2. **右键点击刷新按钮**（地址栏旁边）
3. 选择"**清空缓存并硬性重新加载**"

### 方案 2: 确认前端开发服务器已重启

#### 停止并重启开发服务器：

**步骤 1: 停止当前服务**
- 如果之前运行了 `npm run dev`，按 `Ctrl + C` 停止

**步骤 2: 重新启动开发服务器**
```bash
cd e:\123.207.52.46_8088\new_system\frontend\admin
npm run dev
```

**步骤 3: 等待编译完成**
- 看到 "ready in xxx ms" 或 "Local: http://localhost:5173" 后
- 在浏览器打开或刷新页面

### 方案 3: 验证代码是否正确

打开浏览器开发者工具（F12），在 Console 中执行：

```javascript
// 检查 handleRecognize 函数是否包含日志
console.log('测试日志输出')
```

如果能看到这条日志，说明控制台工作正常。

### 方案 4: 检查文本格式

#### ✅ 正确的文本格式（家教单之间有空行）：

```
S 深圳南山区 初二数学
【学生情况】男孩，数学基础较好
【课费薪酬】180-200/次

S 北京海淀区 高三物理
【学生情况】女孩，物理成绩中等
【课费薪酬】250-300/次

S 上海浦东新区 小五英语
【学生情况】男孩，英语口语较弱
【课费薪酬】120-150/次
```

**关键点**：
- ⚠️ 每个家教单之间**必须有空行**（至少一行完全空白的行）
- ⚠️ 不是单个换行，而是**两个换行符**（形成空行）

#### ❌ 错误的文本格式（没有空行）：

```
S 深圳南山区 初二数学
【学生情况】男孩，数学基础较好
【课费薪酬】180-200/次
S 北京海淀区 高三物理  ← 紧接着上一行，没有空行
【学生情况】女孩，物理成绩中等
【课费薪酬】250-300/次
```

### 方案 5: 测试后端分割逻辑

#### 测试用例：复制以下文本（确保空行存在）

```
深圳南山区 初二数学 150/次

北京海淀区 高三物理 250/次

上海浦东 小五英语 120/次
```

**注意**：
1. 在文本编辑器中准备好文本
2. 确保每个家教单之间有**完全空白的一行**
3. 全选复制后粘贴到识别框

#### 如果还是识别为1个：

**检查是否真的有空行：**
1. 在记事本中打开文本
2. 启用"查看 > 显示符号 > 显示空格和制表符"
3. 确认空行处只有换行符，没有空格

在控制台直接执行这段代码，测试日志是否正常：

```javascript

```

如果能看到这些输出，说明控制台正常，问题出在：
1. 代码没有正确更新（需要强制刷新）
2. 文本格式不正确（没有空行）

## 🎯 常见错误场景

### 场景 1: 看不到任何日志
**原因**：代码没有更新
**解决**：Ctrl + Shift + Delete 清除缓存，然后 Ctrl + F5 强制刷新

### 场景 2: 看到日志但 total: 1
**原因**：文本格式问题，没有空行
**解决**：确保每个家教单之间有完整的空行

### 场景 3: 开发服务器报错
**原因**：依赖问题
**解决**：重新 `npm install`

### 场景 4: 浏览器显示白屏
**原因**：编译错误
**解决**：查看终端的错误信息，修复语法错误
