### 遇到的问题
执行 SQL 添加字段时，phpMyAdmin 报错：
- 错误 #1060: Duplicate column name 'nickname'
- 说明 nickname 字段已经存在

## 解决步骤

### 第1步：检查表结构

在 phpMyAdmin 中执行以下 SQL，查看 users 表的当前结构：

```sql
DESC `users`;
```

**预期结果：**
应该看到类似这样的表结构：

| Field | Type | Null | Key | Default | Extra |
|-------|------|------|-----|---------|-------|
| id | int(11) | NO | PRI | NULL | auto_increment |
| openid | varchar(100) | YES | | NULL | |
| phone | varchar(20) | YES | | NULL | |
| nickname | varchar(100) | YES | | NULL | |
| create_time | datetime | YES | | NULL | |
| update_time | datetime | YES | | NULL | |

**需要确认的字段：**
- ✓ nickname - 如果存在，说明昵称字段已有
- ? avatar - 如果不存在，需要添加
- ? platform - 如果不存在，需要添加

### 第2步：根据表结构添加缺失字段

#### 情况A：如果 avatar 字段不存在

执行：
```sql
ALTER TABLE `users` 
ADD COLUMN `avatar` VARCHAR(500) DEFAULT NULL COMMENT '头像URL' 
AFTER `nickname`;
```

#### 情况B：如果 platform 字段不存在

执行：
```sql
ALTER TABLE `users` 
ADD COLUMN `platform` VARCHAR(20) DEFAULT 'miniprogram' COMMENT '用户端口：miniprogram-小程序, h5-H5端' 
AFTER `avatar`;
```

**注意：** 如果 avatar 不存在，需要先添加 avatar，再添加 platform。

#### 情况C：更新现有数据（必须执行）

```sql
UPDATE `users` 
SET `platform` = 'miniprogram' 
WHERE `platform` IS NULL OR `platform` = '';
```

### 第3步：验证表结构

再次执行：
```sql
DESC `users`;
```

**确认表中包含以下字段：**
- id
- openid
- phone
- nickname ✓
- avatar ✓
- platform ✓
- create_time
- update_time

### 第4步：清空测试数据（可选但推荐）

## 常见问题排查

### 问题1：数据库中 nickname 和 avatar 为 NULL

**可能原因：**
1. 前端没有正确传递参数
2. 后端没有正确接收参数
3. 后端没有正确保存参数

**排查步骤：**

1. 检查前端是否正确获取了头像和昵称：
   - 在小程序 Console 中查看 `this.userNickname` 和 `this.userAvatar` 的值
   - 确认登录请求的参数中包含这两个字段

2. 检查后端是否接收到参数：
   - 查看后端日志 `new_system/backend/runtime/log/`
   - 查找"登录参数"日志，确认 nickname 和 avatar 有值

3. 检查后端是否正确保存：
   - 在 `WechatMiniProgramService.php` 的 `registerOrUpdateUser` 方法中添加日志
   - 确认 $extraInfo 包含 nickname 和 avatar

### 问题2：管理后台看不到用户

**可能原因：**
1. MiniProgramUser 控制器使用了错误的表名
2. 前端 API 请求错误
3. 权限问题

**排查步骤：**

1. 检查控制器是否使用正确的表名：
   - 打开 `new_system/backend/app/controller/admin/MiniProgramUser.php`
   - 确认所有查询都使用 `Db::table('users')`

2. 检查前端 API 请求：
   - 打开浏览器开发者工具的 Network 标签
   - 查看请求是否成功，响应数据是否正确

3. 检查数据库连接：
   - 确认 `.env` 文件中的数据库配置正确
   - 确认数据库用户有权限访问 users 表

### 问题3：phpMyAdmin 报错

**错误类型：**

1. **#1060: Duplicate column name 'xxx'**
   - 说明字段已存在，跳过这条 SQL 即可

2. **#1054: Unknown column 'xxx'**
   - 说明字段不存在，需要先添加该字段

3. **Fatal error: ... resultSetHasJustOneTable()**
   - phpMyAdmin 自身的 bug，与 SQL 语句无关
   - 解决方法：
     - 刷新页面重试
     - 一次只执行一条 SQL
     - 使用命令行工具执行 SQL

**使用命令行执行 SQL：**

如果 phpMyAdmin 持续报错，可以使用命令行：

```bash
# Windows 下打开命令提示符，进入 MySQL 安装目录的 bin 文件夹
cd C:\path\to\mysql\bin

# 连接数据库
mysql -h 127.0.0.1 -P 3306 -u jE2se7DGe5HfE6zL -p myjiajiao

# 输入密码：myjiajiao

# 执行 SQL
DESC users;
ALTER TABLE users ADD COLUMN avatar VARCHAR(500) DEFAULT NULL COMMENT '头像URL' AFTER nickname;
ALTER TABLE users ADD COLUMN platform VARCHAR(20) DEFAULT 'miniprogram' COMMENT '用户端口' AFTER avatar;
UPDATE users SET platform = 'miniprogram' WHERE platform IS NULL OR platform = '';
```
