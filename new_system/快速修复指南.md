# 快速修复指南 - 用户表问题

## 问题
执行 SQL 时 phpMyAdmin 报错：`#1060: Duplicate column name 'nickname'`

## 原因
nickname 字段已经存在，不需要再添加。

## 解决方案（3步搞定）

### 步骤1：在 phpMyAdmin 中执行以下 SQL

打开 phpMyAdmin → 选择 `myjiajiao` 数据库 → 点击 SQL 标签 → 复制粘贴以下 SQL：

```sql
-- 查看表结构
DESC `users`;
```

**查看结果，确认哪些字段已存在：**
- 如果有 `nickname` ✓ - 已存在，不需要添加
- 如果有 `avatar` ✓ - 已存在，不需要添加  
- 如果有 `platform` ✓ - 已存在，不需要添加

### 步骤2：只添加缺失的字段

**如果 avatar 不存在，执行：**
```sql
ALTER TABLE `users` ADD COLUMN `avatar` VARCHAR(500) DEFAULT NULL COMMENT '头像URL';
```

**如果 platform 不存在，执行：**
```sql
ALTER TABLE `users` ADD COLUMN `platform` VARCHAR(20) DEFAULT 'miniprogram' COMMENT '用户端口';
```

**更新现有数据（必须执行）：**
```sql
UPDATE `users` SET `platform` = 'miniprogram' WHERE `platform` IS NULL OR `platform` = '';
```

### 步骤3：验证

```sql
-- 查看表结构，确认字段都在
DESC `users`;

-- 清空测试数据（可选）
TRUNCATE TABLE `users`;
```

## 测试

1. 打开小程序
2. 进入登录页面
3. 点击"用微信头像"，选择头像
4. 输入昵称
5. 同意协议
6. 授权手机号登录

## 检查结果

在 phpMyAdmin 中执行：
```sql
SELECT id, phone, nickname, avatar, platform, create_time 
FROM `users` 
ORDER BY create_time DESC 
LIMIT 5;
```

应该看到：
- phone: 手机号
- nickname: 你输入的昵称
- avatar: 微信头像URL
- platform: 'miniprogram'

## 如果还有问题

### phpMyAdmin 报错怎么办？

**错误 #1060: Duplicate column name**
- 说明字段已存在，跳过这条 SQL

**其他错误**
- 刷新页面重试
- 一次只执行一条 SQL
- 或者使用命令行工具

### 数据还是 NULL 怎么办？

1. 查看后端日志：`new_system/backend/runtime/log/`
2. 查找"登录参数"，确认后端收到了 nickname 和 avatar
3. 如果没收到，检查小程序 Console 的登录请求参数
