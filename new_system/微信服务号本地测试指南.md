# 微信服务号本地测试指南

## 🎯 测试目标

在本地环境测试微信服务号通知功能，验证：
1. AccessToken获取
2. 模板消息发送
3. 通知配置功能
4. 用户OpenID管理

---

## 📋 准备工作

### 方案A：使用微信测试号（推荐）

#### 1. 申请测试号
1. 访问：https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=sandbox/login
2. 用微信扫码登录
3. 记录以下信息：
   - **appID**：`wx1234567890abcdef`
   - **appsecret**：`1234567890abcdef1234567890abcdef`

#### 2. 关注测试号
- 用手机微信扫描页面上的测试号二维码
- 关注后会显示你的 **OpenID**（非常重要，保存下来）
- 示例：`oABC123456789DEFGHIJKLMNOPQRSabc`

#### 3. 添加模板消息
在测试号页面：
1. 找到"模板消息接口"
2. 点击"新增测试模板"
3. 选择或输入模板内容：

**推荐模板1：订单通知**
```
标题：订单通知
内容：
{{first.DATA}}
订单编号：{{keyword1.DATA}}
城市：{{keyword2.DATA}}
区域：{{keyword3.DATA}}
科目：{{keyword4.DATA}}
薪资：{{keyword5.DATA}}
{{remark.DATA}}
```

4. 提交后获得 **模板ID**（类似：`Ab12345678901234567890abcdefghijk`）

## 🔧 系统配置

### 1. 登录管理后台

访问：`http://localhost/admin/`

### 2. 配置微信服务号

进入：**通知配置 > 微信服务号**

填写：
- **AppID**：`wx1234567890abcdef`（从测试号页面复制）
- **AppSecret**：`1234567890abcdef1234567890abcdef`（从测试号页面复制）
- **启用微信通知**：开启
- 点击"保存配置"

### 3. 测试AccessToken

点击"**获取AccessToken**"按钮
- ✅ 成功：提示"AccessToken获取成功"
- ❌ 失败：检查AppID和AppSecret是否正确

### 4. 配置模板消息

进入：**通知配置 > 微信模板**

**方式1：手动添加**
1. 点击"添加模板"
2. 填写：
   - **模板代码**：`order_notify`
   - **模板名称**：`订单通知`
   - **微信模板ID**：`Ab12345678901234567890abcdefghijk`（从测试号页面复制）
   - **字段映射**：
   ```json
   {
     "first": "您有新的家教订单",
     "keyword1": "{{order_no}}",
     "keyword2": "{{city}}",
     "keyword3": "{{district}}",
     "keyword4": "{{subject}}",
     "keyword5": "{{salary}}",
     "remark": "请及时查看处理"
   }
   ```
   - **是否启用**：开启
3. 点击"保存"

**方式2：从微信平台同步（如果配置成功）**
1. 点击"从微信平台同步"
2. 系统自动拉取所有模板
3. 编辑模板配置字段映射

## 🔍 常见问题

### Q1: 提示"获取AccessToken失败"

**原因：**
- AppID或AppSecret错误
- 网络无法访问微信API

**解决：**
1. 重新检查AppID和AppSecret
2. 确保服务器能访问 `api.weixin.qq.com`
3. 检查防火墙设置

### Q2: 提示"用户未关注"或"require subscribe"

**原因：**
- 用户没有关注测试号
- OpenID错误

**解决：**
1. 用手机微信扫码关注测试号
2. 在测试号页面查看正确的OpenID
3. 确保复制完整的OpenID（没有多余空格）

### Q3: 提示"template param error"

**原因：**
- 模板ID错误
- 字段映射配置不正确
- 模板字段与配置不匹配

**解决：**
1. 检查模板ID是否正确
2. 确认字段映射JSON格式正确
3. 字段名称要与微信模板定义一致

### Q4: 提示"api reach limit"

**原因：**
- 接口调用频率超限

**解决：**
- 测试号有频率限制
- 稍等几分钟后重试
- 减少测试频率

### Q5: 收不到消息

**检查清单：**
- [ ] 是否已关注测试号
- [ ] OpenID是否正确
- [ ] 模板ID是否正确
- [ ] 微信通知是否启用
- [ ] 模板是否启用
- [ ] 手机微信是否正常

## 🚀 正式环境准备

测试通过后，要在正式环境使用：

### 1. 申请正式服务号
1. 访问：https://mp.weixin.qq.com/
2. 注册服务号（需企业认证，300元/年）
3. 完成认证流程

### 2. 配置服务器地址
1. 进入：开发 > 基本配置
2. 服务器地址（URL）：`https://你的域名/admin/wechat/callback`
3. Token：自定义（与系统配置一致）
4. 启用服务器配置

### 3. 申请模板消息权限
1. 进入：功能 > 模板消息
2. 选择行业：教育 > 培训
3. 添加或申请模板

### 4. 更新系统配置
- 替换为正式服务号的AppID和AppSecret
- 更新模板ID为正式模板
- 配置正式的服务器地址

## 📝 注意事项

### 测试号限制
- 仅用于开发测试
- 关注用户数有限制（通常100人）
- 某些高级接口可能不可用
- 不建议用于生产环境

### 本地测试限制
- 微信回调无法访问本地地址
- 需要测试回调功能时使用内网穿透工具（如natapp、ngrok）
- 支付相关功能无法在本地完整测试

### 安全建议
- 不要将AppSecret提交到代码仓库
- 测试完成后及时删除测试数据
- 正式环境使用HTTPS
- 定期更换密钥
