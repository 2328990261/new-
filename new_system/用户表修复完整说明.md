# 用户表修复完整说明

## 问题总结
1. User模型使用的是 `wechat_users` 表，但实际数据库中的表名是 `fa_users`
2. 头像和昵称字段可能不存在或字段名不匹配
3. 缺少 `platform` 字段来区分小程序和H5用户

## 已完成的修复

### 1. 修改User模型 (`new_system/backend/app/model/User.php`)
- 将表名从 `wechat_users` 改为 `users`（ThinkPHP会自动加 `fa_` 前缀）
- 更新字段定义，使用 `avatar` 而不是 `headimgurl`
- 添加 `platform` 字段

### 2. 修改后端控制器
- `MiniProgramUser.php`：改为使用 `fa_users` 表
- `WechatMiniProgram.php`：添加调试日志，记录接收到的参数

### 3. 创建SQL文件
- `ensure_users_table_structure.sql`：确保表结构正确的完整SQL

## 执行步骤

### 步骤1：执行SQL确保表结构正确

在phpMyAdmin的SQL标签页中执行以下SQL（一次执行一条，如果报错说字段已存在可以忽略）：

```sql
-- 1. 添加 nickname 字段（如果不存在）
ALTER TABLE `fa_users` ADD COLUMN `nickname` VARCHAR(100) DEFAULT NULL COMMENT '昵称' AFTER `phone`;

-- 2. 添加 avatar 字段（如果不存在）
ALTER TABLE `fa_users` ADD COLUMN `avatar` VARCHAR(500) DEFAULT NULL COMMENT '头像URL' AFTER `nickname`;

-- 3. 添加 platform 字段（如果不存在）
ALTER TABLE `fa_users` ADD COLUMN `platform` VARCHAR(20) DEFAULT 'miniprogram' COMMENT '用户端口：miniprogram-小程序, h5-H5端' AFTER `avatar`;

-- 4. 更新现有数据
UPDATE `fa_users` SET `platform` = 'miniprogram' WHERE `platform` IS NULL OR `platform` = '';
```

### 步骤2：清空现有测试数据（可选）

如果想重新测试，可以清空现有用户数据：

```sql
TRUNCATE TABLE `fa_users`;
```

### 步骤3：重新测试登录

1. 打开小程序
2. 进入登录页面
3. 选择头像（点击"用微信头像"按钮）
4. 输入昵称
5. 同意协议
6. 授权手机号登录

### 步骤4：检查数据是否正确保存

登录成功后，在phpMyAdmin中查看 `fa_users` 表：

```sql
SELECT id, openid, phone, nickname, avatar, platform, create_time 
FROM fa_users 
ORDER BY create_time DESC 
LIMIT 10;
```

应该能看到：
- `phone`: 手机号
- `nickname`: 你输入的昵称
- `avatar`: 微信头像URL
- `platform`: 'miniprogram'

## 常见问题

### Q1: 字段已存在错误
A: 这是正常的，说明字段已经添加过了，可以忽略这个错误。

### Q2: 头像显示为NULL
A: 检查：
1. 前端是否正确获取了微信头像（查看 `this.userAvatar` 的值）
2. 后端日志中是否接收到了 avatar 参数
3. 数据库字段类型是否为 VARCHAR(500)

### Q3: 昵称显示为NULL
A: 检查：
1. 前端是否输入了昵称（查看 `this.userNickname` 的值）
2. 后端日志中是否接收到了 nickname 参数
3. 数据库字段类型是否为 VARCHAR(100)

### Q4: 管理后台看不到用户
A: 检查：
1. `MiniProgramUser.php` 控制器是否使用了正确的表名（应该是 `users`，不是 `wechat_users`）
2. 前端API请求是否正确
3. 浏览器控制台是否有错误信息
