# 表名前缀问题修复方案

## 问题原因

数据库配置中设置了 `'prefix' => 'fa_'`，但实际表名是 `users`（无前缀）。

ThinkPHP 在使用 `Db::name('users')` 时会自动加上前缀，变成 `fa_users`，但数据库中的表名是 `users`，所以查询不到数据。

## 解决方案

将 `users` 表改名为 `fa_users`，与其他表保持一致。

## 执行步骤

### 步骤1：在 phpMyAdmin 中改表名

执行以下 SQL：

```sql
-- 将 users 表改名为 fa_users
RENAME TABLE `users` TO `fa_users`;

-- 验证改名成功
SHOW TABLES LIKE 'fa_users';

-- 查看表结构
DESC `fa_users`;

-- 查看数据
SELECT * FROM `fa_users` LIMIT 5;
```

### 步骤2：验证代码修改

我已经修改了以下文件：

1. **User.php 模型**
   - 注释掉了 `protected $table = 'users';`
   - 现在使用默认规则，ThinkPHP 会自动加 `fa_` 前缀

2. **MiniProgramUser.php 控制器**
   - 将所有 `Db::table('users')` 改为 `Db::name('users')`
   - `Db::name('users')` 会自动加前缀，变成 `fa_users`

### 步骤3：清空测试数据

```sql
TRUNCATE TABLE `fa_users`;
```

### 步骤4：测试小程序登录

1. 打开小程序
2. 进入登录页面
3. 选择头像、输入昵称
4. 授权手机号登录

### 步骤5：验证数据

```sql
SELECT id, phone, nickname, avatar, platform, create_time 
FROM `fa_users` 
ORDER BY create_time DESC 
LIMIT 5;
```

### 步骤6：验证管理后台

打开管理后台的"用户管理"模块，应该能看到用户列表了。

## 为什么要改表名而不是改代码？

1. **保持一致性**：数据库中其他表都有 `fa_` 前缀
2. **减少修改**：只需要改一次表名，不需要修改多处代码
3. **符合规范**：ThinkPHP 的标准做法就是使用表前缀
