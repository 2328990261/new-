# 头像上传系统说明

## 问题背景

之前用户头像显示不出来的问题是因为数据库中存储的是微信小程序的临时文件路径（如 `http://tmp/7jAJD4LeXNEL35e09fb4c4ba5723124fa795d8735519.jpeg`），这些临时文件无法通过HTTP访问。

## 解决方案

已实现完整的头像上传系统，将用户上传的头像保存到服务器实际目录中。

### 1. 后端实现

#### 头像上传控制器 (`AvatarUpload.php`)
- 接收小程序上传的头像文件
- 验证文件类型和大小（最大2MB）
- 保存到服务器目录：`uploads/avatars/年月/`
- 更新数据库中的头像URL

#### 存储路径结构
```
开发环境：tjiajiao91-main/new_system/backend/public/uploads/avatars/202603/avatar_xxx.jpg
线上环境：/www/wwwroot/域名/uploads/avatars/202603/avatar_xxx.jpg
```

#### 访问URL
```
开发环境：http://localhost:8000/uploads/avatars/202603/avatar_xxx.jpg
线上环境：https://域名/uploads/avatars/202603/avatar_xxx.jpg
```

### 2. 前端实现

#### 小程序端 (`pages/profile/index.vue`)
- 使用 `button` 的 `open-type="chooseAvatar"` 选择头像
- 通过 `uni.uploadFile()` 上传到服务器
- 更新本地存储和界面显示

#### 管理端 (`MiniUserManage.vue`)
- 显示用户头像列表
- 处理头像加载失败的情况
- 对无效的临时URL显示默认头像

### 3. 数据库更新

头像上传成功后会同时更新两个表：
- `fa_users.avatar` - 用户表头像字段
- `fa_wechat_users.headimgurl` - 微信用户表头像字段

### 4. 环境适配

#### 开发环境
- 返回完整URL：`http://localhost:8000/uploads/avatars/202603/avatar_xxx.jpg`

#### 线上环境
- 返回相对路径：`uploads/avatars/202603/avatar_xxx.jpg`
- 前端自动拼接域名

### 5. 错误处理

- 文件类型验证：只允许 jpg, jpeg, png, gif, webp
- 文件大小限制：最大2MB
- 临时文件过滤：自动识别并忽略微信临时文件路径
- 加载失败处理：显示默认头像

## 使用说明

### 用户操作流程
1. 在小程序个人中心点击头像
2. 选择新头像
3. 系统自动上传到服务器
4. 更新显示新头像

### 管理员查看
1. 登录管理后台
2. 进入"小程序用户"页面
3. 查看用户头像列表
4. 点击"查看"可查看用户详情

## 注意事项

1. **文件权限**：确保线上环境 `uploads/avatars/` 目录有写入权限
2. **域名配置**：线上环境需要正确配置域名
3. **HTTPS**：线上环境建议使用HTTPS协议
4. **清理旧数据**：可以考虑清理数据库中的临时URL记录

## 测试建议

1. 在小程序中上传新头像
2. 检查服务器目录是否生成文件
3. 在管理后台查看头像是否正常显示
4. 测试不同环境下的URL生成
