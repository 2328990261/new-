# 微信分享卡片配置指南

## 问题原因

你的H5支付页面微信分享无法显示卡片形式的主要原因：

1. **分享图片为空** - 后端返回的 `image` 字段为空字符串
2. **默认图片不合适** - 前端使用 `favicon.ico` 作为默认图片，尺寸太小

## 已完成的代码修改

✅ 修改了后端 `Wechat.php`，添加分享图片配置支持
✅ 修改了前端 `wechatShare.js`，更新默认分享图片路径
✅ 修改了 `Payment.vue`，使用后端配置的分享图片

## 需要你完成的步骤

### 1. 准备分享图片

创建一张符合微信要求的分享图片：

- **尺寸**：建议 300x300 像素或更大（正方形最佳）
- **格式**：JPG 或 PNG
- **大小**：不超过 32KB（建议）
- **内容**：包含你的品牌 logo 和简短文字说明

### 2. 上传分享图片

将准备好的图片上传到以下位置之一：

**选项A：放在项目静态资源目录**
```
tjiajiao91-main/new_system/frontend/user/public/static/images/share-logo.png
```

**选项B：使用现有的上传目录**
```
tjiajiao91-main/new_system/backend/public/uploads/share-logo.png
```

### 3. 配置数据库（可选）

如果你想在数据库中配置分享图片，可以在 `fa_notification_config` 表中添加字段：

```sql
-- 添加微信分享图片字段
ALTER TABLE `fa_notification_config` 
ADD COLUMN `wechat_share_image` VARCHAR(500) DEFAULT '' COMMENT '微信分享图片URL' 
AFTER `wechat_callback_domain`;

-- 更新分享图片URL（替换为你的实际图片URL）
UPDATE `fa_notification_config` 
SET `wechat_share_image` = 'https://你的域名/static/images/share-logo.png' 
WHERE `id` = 1;
```

### 4. 验证配置

#### 4.1 检查图片是否可访问

在浏览器中访问：
```
https://你的域名/static/images/share-logo.png
```

确保图片能正常显示。

#### 4.2 检查微信分享配置接口

在浏览器中访问：
```
https://你的域名/api/wechat/share-config?url=https://你的域名
```

检查返回的 JSON 中 `data.image` 字段是否有值。

#### 4.3 在微信中测试分享

1. 在微信中打开支付页面
2. 点击右上角 "..." 菜单
3. 选择"发送给朋友"或"分享到朋友圈"
4. 查看是否显示卡片形式，包含图片、标题和描述

### 5. 调试技巧

如果分享仍然不显示卡片，可以：

#### 5.1 开启微信调试模式

在URL后添加参数：
```
https://你的域名/payment?wxdebug=1
```

这会在微信开发者工具中显示详细的错误信息。

#### 5.2 检查 JSSDK 配置

在微信中打开页面，按 F12 打开控制台，查看是否有错误信息：
- `config:invalid signature` - 签名错误，检查域名配置
- `config:invalid url domain` - 域名未在微信公众平台配置

#### 5.3 验证微信公众平台配置

登录微信公众平台 → 设置与开发 → 公众号设置 → 功能设置：

- **JS接口安全域名**：必须配置你的域名（不含 http:// 和路径）
- **网页授权域名**：必须配置你的域名

## 常见问题

### Q1: 分享时只显示链接，没有卡片？

**原因**：
- 图片URL无效或无法访问
- 图片尺寸不符合要求
- 微信缓存了旧的分享信息

**解决**：
- 确保图片URL可以公网访问
- 使用 300x300 以上的图片
- 更换分享链接的参数（如添加 `?v=2`）清除缓存

### Q2: 分享卡片显示，但图片是默认图标？

**原因**：
- 图片加载失败
- 图片格式不支持
- 图片大小超过限制

**解决**：
- 检查图片URL是否正确
- 使用 JPG 或 PNG 格式
- 压缩图片到 32KB 以下

### Q3: iOS 和 Android 表现不一致？

**原因**：
- iOS 微信对 JSSDK 签名URL有特殊要求

**解决**：
- 代码已处理 iOS 兼容性（使用首次进入的URL签名）
- 确保不要在页面中使用 `location.href` 改变URL
