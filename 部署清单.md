# 部署清单

## 📦 需要上传的文件

### 1. 管理后台前端
- **源目录**: `tjiajiao91-main/new_system/frontend/admin/dist`
- **目标位置**: 服务器 `/var/www/admin` 或你的管理后台目录
- **访问地址**: `https://admin.your-domain.com`

### 2. 用户端前端
- **源目录**: `tjiajiao91-main/new_system/frontend/user/dist`
- **目标位置**: 服务器 `/var/www/user` 或你的用户端目录
- **访问地址**: `https://www.your-domain.com`

### 3. 后端
- **源目录**: `tjiajiao91-main/new_system/backend`（整个目录）
- **目标位置**: 服务器 `/var/www/backend` 或你的后端目录
- **访问地址**: `https://api.your-domain.com`

---

## ✅ 部署步骤

### 步骤1: 本地打包
```bash
# 双击运行
打包脚本.bat
```

### 步骤2: 上传文件
使用FTP/SFTP工具（如FileZilla）上传以下内容：
- [ ] 管理后台 dist 目录
- [ ] 用户端 dist 目录  
- [ ] 后端整个 backend 目录

### 步骤3: 配置后端
1. [ ] 编辑 `backend/.env` 文件，配置数据库信息
2. [ ] 设置目录权限：`chmod -R 777 backend/runtime`
3. [ ] 确保 `backend/public` 是Web根目录

### 步骤4: 配置Nginx/Apache
1. [ ] 配置管理后台虚拟主机
2. [ ] 配置用户端虚拟主机
3. [ ] 配置后端API虚拟主机
4. [ ] 重启Web服务器

### 步骤5: 数据库更新（如果需要）
```bash
# 添加邀请码字段
mysql -u用户名 -p数据库名 < add_invitation_code_field.sql

# 创建邀请系统表（如果需要启用邀请功能）
mysql -u用户名 -p数据库名 < fix_invitation_tables.sql
```

### 步骤6: 验证部署
- [ ] 访问管理后台，测试登录
- [ ] 访问用户端，测试页面加载
- [ ] 测试API接口是否正常
- [ ] 检查小程序是否能正常连接

## 🔧 重要配置文件

### 后端配置 (backend/.env)
```env
APP_DEBUG = false

[DATABASE]
TYPE = mysql
HOSTNAME = 你的数据库地址
DATABASE = 数据库名
USERNAME = 数据库用户名
PASSWORD = 数据库密码
HOSTPORT = 3306
```

### 前端API地址
- 管理后台: 在打包时已配置
- 用户端: 在打包时已配置
- 小程序: 自动根据环境切换（开发版用localhost，正式版用线上地址）

## 📝 注意事项

1. **备份数据库**：部署前务必备份现有数据库
2. **测试环境**：建议先在测试环境部署验证
3. **权限设置**：确保runtime目录有写入权限
4. **HTTPS配置**：生产环境建议配置SSL证书
5. **缓存清理**：部署后清理浏览器缓存

---

## 🚨 常见问题

### 问题1: 前端页面空白
**解决**: 检查前端打包时的base路径配置

### 问题2: API请求失败
**解决**: 
- 检查后端.env配置
- 检查Nginx/Apache配置
- 查看后端runtime/log日志

### 问题3: 数据库连接失败
**解决**: 
- 检查数据库地址、用户名、密码
- 确认数据库服务已启动
- 检查防火墙设置

---

## 📞 技术支持

如有问题，请检查：
1. 后端日志: `backend/runtime/log/`
2. Nginx日志: `/var/log/nginx/error.log`
3. PHP错误日志: `/var/log/php-fpm/error.log`
